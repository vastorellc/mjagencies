## Overview

Multi-user platform for short-form video content creators. Upload video → in-browser analysis → AI copy generation → auto-upload to YouTube/Instagram/Facebook. Separate Content Research Engine for trend-based content ideation. Admin panel for platform management. Built for Pakistani creators across travel, hotels, car/bike drives, coding, and lifestyle niches.

**Architecture:** Supabase (Auth + PostgreSQL + Realtime) + VPS (Node.js + Nginx + file storage) + pg-boss (PostgreSQL-backed job queue). Every user account created by admin only — no public registration.

## Milestone Summary

**Key Accomplishments:**

1. **Auth-gated multi-user platform** — Supabase Auth + RLS per-user isolation, Express double-gated middleware, admin-only account creation, no public signup
2. **In-browser video analysis engine** — FFmpeg.wasm + TF.js + Web Audio + Canvas, zero server-side video processing cost, 337/340 tests passing
3. **Multi-provider AI copy generation** — Gemini Files API, Claude browser, OpenAI backend proxy, DeepSeek — one call returns 5 platform cards with "Get Better Version" second pass
4. **Auto-upload to YouTube + Instagram + Facebook** — Resumable YouTube, container-in-job Meta, PKT peak-time scheduling via pg-boss, Realtime status push
5. **Learning loops + Content Research Engine** — EMA score calibration, hashtag/hook ranking from user history, external trend data from 4 sources, AI-generated content briefs with 7-day calendar
6. **Admin panel + Provider health monitoring** — 5-tab admin (queue/users/health/logs/stats), provider model verification with weekly health-check job, cover-frame scoring

**Key Decisions:**
- Supabase (Auth + PostgreSQL + Realtime) over raw PostgreSQL + Redis — RLS solves per-user isolation at DB level
- pg-boss over BullMQ + Redis — queue lives in Supabase DB, no extra service needed
- VPS file storage over Supabase Storage — cheaper, user data control, files are temporary
- In-browser video analysis — zero per-video API cost regardless of user count
- Engine v3 (HTMLVideoElement + rVFC) over ffmpeg.wasm for frame extraction — eliminated WASM dependency for core frame pipeline
- Content Research Engine as separate screen — research is pre-production, upload is post-production workflow

**Technical Debt Incurred:**
- Phase 2: Real OAuth E2E round-trips deferred pending live Google/Meta credentials
- Phase 3: Engine v3 works but uses HTMLVideoElement — ffmpeg.wasm scene detection still available as fallback
- Phase 6: TikTok gated behind TIKTOK_APPROVED=true flag — code written, needs API approval
- Phase 11: DeepSeek vision unsupported (model limitation, vision: false in capability matrix)
- No E2E test suite (0 Playwright tests) — all testing is Vitest unit/integration
- REQUIREMENTS.md ~5 checkboxes need update (documentation debt only — all code is functionally complete)

**Deferred Items:**
- Social login (Google/Meta for signup) — v2
- X/Twitter auto-upload — v2
- Bulk video processing — v2
- Export history CSV — v2
- Competitor channel tracking — v2
