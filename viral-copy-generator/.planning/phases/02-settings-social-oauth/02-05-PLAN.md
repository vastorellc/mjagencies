<objective>
Implement the weekly Meta long-lived-token refresh job (SETTINGS-07). Mondays 09:00 UTC the job iterates every user with an Instagram connection, calls `refreshInstagramToken` on the decrypted token, encrypts the rotated token, and JSONB-merges it back into `platform_config.instagram`. Per-user errors logged but never cascade.

Why weekly when tokens last 60 days: 7-day cadence gives an 8.5x safety margin (research Pitfall 6). Meta has NO refresh-token concept; missing the refresh window forces user re-auth, breaking Phase 6 uploads. Facebook page_access_tokens have a long expiry tied to the user token and are NOT refreshed by this job (research Pattern 7 — Instagram only).

Output: `meta-refresh.ts` with the queue registration + worker body, integration into `index.ts` startup, and tests covering registration, per-user iteration, error isolation, and encryption round-trip.
</objective>

**Step 1** — Create `backend/src/lib/meta-refresh.ts`:

```typescript
import { PgBoss } from 'pg-boss'
import { eq, sql } from 'drizzle-orm'
import { db } from '../db/index.js'
import { settings, type PlatformConfig } from '../db/schema.js'
import { encrypt, decrypt } from './encryption.js'
import { refreshInstagramToken } from './oauth-meta.js'

const QUEUE_NAME = 'meta-token-refresh'
const CRON = '0 9 * * 1' // Mondays 09:00 UTC — 7-day cadence vs. 60-day token (8.5x safety margin)

export async function registerMetaTokenRefreshJob(boss: PgBoss): Promise&lt;void&gt; {
  // Pitfall 5: createQueue must precede schedule (v12 FK on pgboss.schedule.name)
  await boss.createQueue(QUEUE_NAME)

  // schedule() is not idempotent — duplicate is fine on restart (same Phase 1 pattern)
  try {
    await boss.schedule(QUEUE_NAME, CRON, {})
  } catch (err: unknown) {
    const msg = (err as Error).message ?? ''
    if (!msg.includes('duplicate') &amp;&amp; !msg.includes('unique')) throw err
  }

  await boss.work(QUEUE_NAME, async (_job) =&gt; {
    await refreshAllInstagramTokens()
    console.log('[pg-boss] meta-token-refresh completed')
  })

  console.log('[pg-boss] meta-token-refresh job registered (cron:', CRON, ')')
}

interface SettingsRow {
  user_id: string
  platform_config: PlatformConfig | null
}

/**
 * Worker body — exported separately so unit tests can call it without going through pg-boss.
 * Iterates every user whose Instagram is connected, refreshes the long-lived token,
 * encrypts and merges the new value. Per-user errors logged but never abort the loop.
 */
export async function refreshAllInstagramTokens(): Promise&lt;void&gt; {
  // Only rows where instagram is non-null (Facebook tokens are NOT refreshed here — research Pattern 7)
  const rows = await db.execute&lt;SettingsRow&gt;(sql`
    SELECT user_id, platform_config
    FROM settings
    WHERE platform_config -&gt; 'instagram' IS NOT NULL
      AND platform_config -&gt; 'instagram' != 'null'::jsonb
  `)
  const data = (rows as unknown as { rows: SettingsRow[] }).rows ?? (rows as unknown as SettingsRow[])

  for (const row of data) {
    const ig = row.platform_config?.instagram
    if (!ig?.access_token) continue
    try {
      const plaintext = decrypt(ig.access_token)
      const refreshed = await refreshInstagramToken(plaintext)
      const patch = {
        instagram: {
          access_token: encrypt(refreshed.access_token),
          expiry: Date.now() + refreshed.expires_in * 1000,
        },
      }
      await db.transaction(async (tx) =&gt; {
        await tx.execute(sql`SELECT id FROM settings WHERE user_id = ${row.user_id} FOR UPDATE`)
        await tx
          .update(settings)
          .set({
            platform_config: sql`COALESCE(${settings.platform_config}, '{}')::jsonb || ${JSON.stringify(patch)}::jsonb`,
            updated_at: sql`NOW()`,
          })
          .where(eq(settings.user_id, row.user_id))
      })
      console.log(`[meta-token-refresh] refreshed user ${row.user_id}`)
    } catch (err) {
      // Pitfall 6: log per-user failure, do NOT propagate
      console.error(`[meta-token-refresh] FAILED user ${row.user_id}: ${(err as Error).message}`)
    }
  }
}
```

| Stored ciphertext → memory plaintext (worker context) | Only inside the worker; never returned to a client |
| Worker → graph.instagram.com | Trusted (TLS + provider authenticity); response shape validated |
| Refreshed plaintext → DB | Re-encrypted before crossing |
