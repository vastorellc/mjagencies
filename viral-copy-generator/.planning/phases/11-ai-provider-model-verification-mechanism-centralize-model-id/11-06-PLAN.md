| AdminPage→backend | GET /api/admin/provider-health gated by adminMiddleware (existing Phase 8) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-11-18 | Information disclosure | capability badges visible to all users | accept | Capabilities are public model metadata (no commercial confidentiality); same data published on provider docs |
| T-11-19 | Tampering | model_id dropdown bypass via DevTools | mitigate | Backend whitelist check (Plan 03 Task 2) re-validates model_id against MODELS — dropdown is UX only, server is authoritative |
| T-11-20 | XSS | error_message rendered in banner | mitigate | React's default escaping handles this; do NOT use dangerouslySetInnerHTML; error_message is plain text only |
| T-11-21 | Spoofing | admin sees fabricated health data | mitigate | Data sourced exclusively from admin_provider_health table written by pg-boss worker; no user input path can mutate it |
