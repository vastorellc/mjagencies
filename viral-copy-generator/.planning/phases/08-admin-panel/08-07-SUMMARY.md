## Decisions Made

- JOB_STATE_STYLES uses `??` fallback for unknown job states — pg-boss may add new states in future versions; the fallback `'bg-zinc-700 text-zinc-300'` prevents render failure for unrecognized states
- showAllJobs checkbox matches the `includeAll` param design from 08-06 — default hides cancelled/completed jobs so the queue view focuses on actionable states
- Inline IIFE `{platformStats.length > 0 && (() => { ... })()}` for stats render — avoids let/const declarations at JSX expression level while keeping maxUploads calculation co-located with the render loop
