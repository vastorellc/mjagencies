## Spacing Scale

Declared values (all multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Inline padding (error badge px) |
| sm | 8px | Error badge vertical padding (py-2) |
| md | 16px | Form field horizontal padding (px-4), page horizontal padding (px-4) |
| lg | 24px | — (not used in Phase 1) |
| xl | 32px | — (not used in Phase 1) |
| 2xl | 48px | — (not used in Phase 1) |
| 3xl | 64px | — (not used in Phase 1) |

Form internal spacing: `space-y-4` (16px between all form children — heading, error, email input, password input, submit button).

Exceptions: Touch targets. The submit button uses `py-3` (12px vertical padding). At 16px font this gives ~44px total height — meets iOS minimum touch target. This is intentional and is the only non-scale exception.

Source: 01-RESEARCH.md Pattern 8 (LoginPage.tsx), CLAUDE.md UI-05.

|------|-------|----------------|-------|

Accent reserved for:
1. Submit button background (`bg-purple-600`)
2. Submit button hover (`hover:bg-purple-500`)

Accent is NOT used for: text, borders, layout, headings, labels. Three elements only.

Placeholder text: zinc-400 (`#a1a1aa`) — serves as muted secondary on input fields.

## Component Inventory

### LoginPage

Single file: `frontend/src/pages/LoginPage.tsx`

**Layout:**
- Outer container: `flex h-[100dvh] items-center justify-center bg-zinc-950 px-4`
  - `h-[100dvh]` is mandatory — not `h-screen` (iOS Safari viewport bug, CLAUDE.md UI-05)
- Form: `w-full max-w-sm space-y-4`

**Elements (top to bottom):**

1. Heading
   - Tag: `<h1>`
   - Classes: `text-2xl font-bold text-white`
   - Copy: "Viral Copy Generator"

2. Error banner (conditional, shown only when `error !== null`)
   - Tag: `<p>`
   - Classes: `rounded bg-red-900/40 px-3 py-2 text-sm text-red-300`
   - Content: Supabase error message string (e.g. "Invalid login credentials")

3. Email input
   - Tag: `<input type="email">`
   - Classes: `w-full rounded-lg bg-zinc-800 px-4 py-3 text-white placeholder-zinc-400 outline-none focus:ring-2 focus:ring-purple-500`
   - Placeholder: "Email"
   - Attributes: `required`

4. Password input
   - Tag: `<input type="password">`
   - Classes: `w-full rounded-lg bg-zinc-800 px-4 py-3 text-white placeholder-zinc-400 outline-none focus:ring-2 focus:ring-purple-500`
   - Placeholder: "Password"
   - Attributes: `required`

5. Submit button
   - Tag: `<button type="submit">`
   - Classes: `w-full rounded-lg bg-purple-600 py-3 font-semibold text-white transition hover:bg-purple-500 disabled:opacity-50`
   - States:
     - Default: "Sign in"
     - Loading: "Signing in..." + `disabled` attribute applied
   - Disabled visual: `opacity-50` — button dims but retains shape and color

**States the component must handle:**

| State | Trigger | Visual |
|-------|---------|--------|
| Idle | Page load / error cleared | No banner, inputs empty, button reads "Sign in" |
| Loading | Form submitted, awaiting Supabase | Button reads "Signing in...", `disabled:opacity-50` applied |
| Error | Supabase returns error | Red banner above first input with error message text |
| Success | SIGNED_IN event fires | Component unmounts — App renders next screen |

No skeleton state. No success toast. Auth state change is handled by `onAuthStateChange` in App.tsx — LoginPage does not navigate itself.

### App.tsx Auth Gate

`frontend/src/App.tsx`

**Loading state (session check in-flight):** `return null` — blank white frame for <100ms. No spinner in Phase 1.

**Unauthenticated state:** `return <LoginPage />`

**Authenticated state:** `return <GeneratorPage />` — placeholder component, renders nothing until Phase 3.

Source: 01-RESEARCH.md Pattern 8 (App.tsx).

## Interaction Contract

| Interaction | Behavior |
|-------------|----------|
| Form submit | `e.preventDefault()` → `setLoading(true)` → `supabase.auth.signInWithPassword({ email, password })` |
| Success | `onAuthStateChange` fires SIGNED_IN → App re-renders with session → LoginPage unmounts |
| Error | `setError(error.message)` → `setLoading(false)` → error banner renders above inputs |
| Tab order | Natural DOM order: email → password → button |
| Keyboard | Enter in password field submits form (native `<form>` behavior) |
| Direct URL navigation | No routes exist. App.tsx renders LoginPage for any unauthenticated state regardless of URL. |

## Viewport Contract

The viewport meta tag in `frontend/index.html` must include:

```html
<meta name="viewport" content="width=device-width, initial-scale=1.0, viewport-fit=cover" />
```

`viewport-fit=cover` is mandatory for iOS Safari safe area handling (CLAUDE.md UI-05).
`h-[100dvh]` is mandatory on the login container — not `h-screen`.
