# Phase 5 — UI Design Contract

> Visual and interaction contract for Phase 5: AI Copy + Platform Cards.
> Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none |
| Preset | not applicable |
| Component library | none |
| Icon library | none (Unicode characters and SVG inline where required) |
| Font | System font stack (Tailwind default) |

**Source:** CLAUDE.md §"Frontend" — "No UI component library — Tailwind CSS only."
Tailwind v4 via `@import "tailwindcss"` + `@theme {}` in `frontend/src/styles.css`.
No `tailwind.config.js` — all custom tokens declared in `@theme {}` block.

**Critical Tailwind rule (from STATE.md ScorePanel pattern):**
All Tailwind class strings MUST be full static strings — no dynamic string interpolation or
template literals for class names. The JIT scanner sees only literal text. Store per-variant
class strings in `Record<Key, string>` maps (same pattern as `BAND_CLASSES` in ScorePanel.tsx
and `PLATFORM_META` in PlatformCardGrid.tsx).

## Spacing Scale

Standard 8-point scale. All values are multiples of 4.

| Token | Tailwind class | Value | Usage |
|-------|---------------|-------|-------|
| xs | `gap-1`, `p-1` | 4px | Icon-to-label gaps, status dot spacing |
| sm | `gap-2`, `p-2` | 8px | Inline copy-button padding, field row gap |
| md | `gap-4`, `p-4` | 16px | Card internal padding (default), main content gap |
| lg | `gap-6`, `px-4 py-3` | 24px | Section padding between card stack items |
| xl | `gap-8` | 32px | Page-level stack gap between card stack and score panel |
| 2xl | `py-12` | 48px | Empty-state vertical centering padding |
| 3xl | — | 64px | Not used in Phase 5 |

Exceptions:
- Copy button touch target: minimum `min-h-[44px]` on mobile (iOS HIG touch target standard)
- File picker label: `px-4 py-3` (16px/12px) to match existing form inputs in LoginPage

## Typography

Established pattern from LoginPage.tsx and existing components. Phase 5 adds one type size for TikTok hook prominence.

| Role | Size | Weight | Line Height | Tailwind classes |
|------|------|--------|-------------|-----------------|
| Label | 12px | 400 (normal) | 1.4 | `text-xs` |
| Body | 14px | 400 (normal) | 1.5 | `text-sm` (default leading) |
| Platform Hook | 16px | 700 (bold) | 1.4 | `text-base font-bold` |
| App title / H1 | 24px | 700 (bold) | 1.2 | `text-2xl font-bold` |

Weight rule (locked in STATE.md §"Accumulated Context"):
- Two weights only: `font-normal` (400) for body text and `font-bold` (700) for headings/buttons.
- `font-semibold` is NOT used — overridden by UI-SPEC checker per STATE.md note.

Character count display: always `text-xs text-zinc-400` right-aligned inside each field row.
Format: `{current}/{limit}` — e.g., `47/60`, `182/200`, `143/150`.
When count exceeds limit: switch to `text-xs text-red-400` to signal overflow.

## Platform Card Specs

### Component: `PlatformCopyCard`

One component handles all 5 platforms. Platform-specific rendering is driven by a
`PLATFORM_CONFIG` Record (full static class strings — same pattern as `BAND_CLASSES`).

**Card wrapper** (per platform, full static class string):

```
// Store as Record<Platform, string> — never interpolate
youtube:   "rounded-lg border-2 border-red-500 bg-red-950 p-4 flex flex-col gap-3"
instagram: "rounded-lg border-2 border-pink-500 bg-pink-950 p-4 flex flex-col gap-3"
tiktok:    "rounded-lg border-2 border-cyan-400 bg-zinc-900 p-4 flex flex-col gap-3"
facebook:  "rounded-lg border-2 border-blue-500 bg-blue-950 p-4 flex flex-col gap-3"
x:         "rounded-lg border-2 border-zinc-400 bg-zinc-900 p-4 flex flex-col gap-3"
```

**Card header row:**
```
<div class="flex items-center justify-between">
  <span class="text-sm font-bold text-white">{Platform label}</span>
  {Upload button | TikTok disabled | nothing for X}
</div>
```

**Field row layout** (each copyable field):
```
<div class="flex flex-col gap-1">
  <div class="flex items-center justify-between">
    <span class="text-xs text-zinc-400">{Field label}</span>
    <div class="flex items-center gap-2">
      <span class="text-xs text-zinc-500">{char count}</span>  // if applicable
      <button class="rounded bg-zinc-700 px-2 py-1 text-xs hover:bg-zinc-600 min-h-[32px]">
        Copy | Copied!
      </button>
    </div>
  </div>
  <p class="text-sm text-zinc-100 whitespace-pre-wrap">{field content}</p>
</div>
```

**Tags/hashtags field** (array rendered as space-separated string for copy):
```
<p class="text-sm text-zinc-100 flex flex-wrap gap-1">
  {tags.map(tag => <span class="rounded bg-zinc-800 px-2 py-1 text-xs">{tag}</span>)}
</p>
```
Copy button copies `tags.join(' ')` (space-separated) for YouTube tags; `hashtags.join(' ')` for hashtag fields.

**Character count overflow:**
- Within limit: `text-xs text-zinc-500`
- Exceeds limit: `text-xs text-red-400`

**Field dividers:** `<hr class="border-zinc-700">` between fields within a card.

## Interaction States

### Generate Copy Button

| Condition | State | Classes |
|-----------|-------|---------|
| No file AND description empty | Disabled | `w-full rounded-lg bg-purple-600 py-3 font-bold text-white opacity-50 cursor-not-allowed` |
| File selected OR description non-empty, no prior generation | Enabled | `w-full rounded-lg bg-purple-600 py-3 font-bold text-white transition hover:bg-purple-500` |
| File selected OR description non-empty, after first generation | Enabled ("Regenerate") | Same enabled classes |
| AI call in flight | Loading | `w-full rounded-lg bg-purple-600 py-3 font-bold text-white opacity-75 cursor-not-allowed` + spinner + "Generating copy…" |

Spinner element: `<span class="inline-block h-4 w-4 animate-spin rounded-full border-2 border-white border-t-transparent mr-2">` (inline before button text).

### Get Better Version Button

Rendered only after first successful AI generation. Positioned directly below the X card (last card), before the ScorePanel.

```
<button class="w-full rounded-lg border border-zinc-600 bg-zinc-800 py-3 text-sm text-zinc-200 transition hover:bg-zinc-700">
  Get Better Version
</button>
```

Loading state (second call in flight): same spinner pattern as Generate Copy button + "Improving copy…" text.

### Copy Button Flash (1.5s)

On click:
1. Immediately: change button text from "Copy" to "Copied!" and add `text-green-400` to the button text class.
2. After 1500ms: reset text to "Copy" and remove `text-green-400`.

Button during feedback: `rounded bg-zinc-700 px-2 py-1 text-xs text-green-400 hover:bg-zinc-600 min-h-[32px]`
Button default: `rounded bg-zinc-700 px-2 py-1 text-xs text-zinc-200 hover:bg-zinc-600 min-h-[32px]`

Implementation: `useState<Record<string, boolean>>` keyed by `"{platform}-{fieldName}"`. `setTimeout` of 1500ms to reset.

### Upload Button States (per card, driven by Supabase Realtime)

| State | Classes | Label |
|-------|---------|-------|
| idle | `rounded bg-zinc-700 px-3 py-1 text-xs text-zinc-200 hover:bg-zinc-600` | "Upload" |
| uploading | `rounded bg-zinc-700 px-3 py-1 text-xs text-zinc-400 cursor-not-allowed flex items-center gap-1` | Spinner + "Uploading…" |
| posted | `rounded bg-green-800 px-3 py-1 text-xs text-green-200 cursor-default` | "Posted ✓" |
| failed | `rounded bg-red-900 px-3 py-1 text-xs text-red-300 hover:bg-red-800` | "Retry Upload" |

TikTok upload (always):
`rounded bg-zinc-800 px-3 py-1 text-xs text-zinc-500 cursor-not-allowed`
Label: "Available once API approved"

X: no upload button rendered.

## Accessibility Notes

- All icon-only buttons (upload state icons) must have `aria-label` with state-specific text:
  - uploading: `aria-label="Uploading to {platform}"`
  - posted: `aria-label="Posted to {platform}"`
  - failed: `aria-label="Upload to {platform} failed — click to retry"`
- Copy button: `aria-label="Copy {field label} for {platform}"` (before copy), `aria-live="polite"` region for "Copied!" feedback.
- File input: hidden visually (`class="sr-only"`), wrapped in `<label>` so clicking the styled div activates it.
- Disabled buttons: `disabled` attribute present (not just visual `opacity-50`) for screen readers.
- TikTok upload button: `disabled` attribute + `title="TikTok API approval pending"`.
- Description textarea: `aria-label="Optional description to help AI generate better copy"`.
