1. **Bug 1 — `extractFrames` hangs without log/timeout/clamp.** On a 22 MB real video, `meta.totalFrames` from ffprobe came back as 0 or NaN, so `N = Math.max(1, Math.floor(totalFrames / 10))` collapsed to `N=1`, telling ffmpeg to emit one JPEG per source frame into MEMFS — runaway disk-in-WASM with no log listener, no timeout, no progress.

2. **Bug 2 — Cancel button does not abort the engine.** `GeneratorPage.onCancel()` only mutates React state. `analyse()` keeps running, the FFmpeg singleton stays locked inside `ff.exec()`, and every subsequent Analyse click queues behind the hung call forever.

<!-- Abort contract -->

- `analyse(file, { signal })` — if `signal.aborted` is already true on entry, throw `new DOMException('aborted', 'AbortError')` immediately.
- Between every internal stage (`probeVideo`, `detectScenes`, `extractFrames`, decode loop, `detectFacesAcrossFrames`, `detectObjectsAndMotion`, `analyseAudio`, `computeBrightnessAcrossFrames`), the orchestrator checks `signal?.aborted` and throws `AbortError` on true.
- On abort (anywhere in the run), the orchestrator's `catch` branch:
  1. Calls `ff.terminate()` (FFmpeg API — kills the WASM worker).
  2. Sets `ffmpegInstance = null` and `ffmpegLoadPromise = null` so the next `getFFmpeg()` rebuilds.
  3. Re-throws the AbortError so the caller can branch.
- The existing `finally` block in `analyse()` already does `safeDelete(ff, [...])`. After terminate, `safeDelete` will throw — wrap each `safeDelete` call in try/catch so cleanup attempts are best-effort and don't mask the AbortError.

    ```typescript
    async function extractFrames(ff: FFmpeg, meta: ParsedMeta): Promise<string[]> {
      // Derive a safe N. If totalFrames is present and >= FRAME_TARGET, sample every (totalFrames/FRAME_TARGET).
      // Otherwise fall back to duration*fps; if both are missing, use FALLBACK_DURATION_SEC * FALLBACK_FPS.
      const estTotal =
        meta.totalFrames >= FRAME_TARGET
          ? meta.totalFrames
          : (meta.durationSec > 0 && meta.fps > 0
              ? Math.floor(meta.durationSec * meta.fps)
              : FRAME_FALLBACK_DURATION_SEC * FRAME_FALLBACK_FPS)
      const N = Math.max(1, Math.floor(estTotal / FRAME_TARGET))

      const onLog = ({ message }: { message: string }) => {
        // eslint-disable-next-line no-console
        console.info(`[ffmpeg:frames] ${message}`)
      }
      ff.on('log', onLog)

      let timeoutId: ReturnType<typeof setTimeout> | null = null
      try {
        const exec = ff.exec([
          '-i', 'input.mp4',
          '-vf', `select='not(mod(n\\,${N}))',scale=512:512:force_original_aspect_ratio=decrease,pad=512:512:(ow-iw)/2:(oh-ih)/2`,
          '-frames:v', String(FRAME_TARGET),
          '-vsync', 'vfr', '-q:v', '5', 'frame_%03d.jpg',
        ])
        const timeout = new Promise<never>((_, reject) => {
          timeoutId = setTimeout(
            () => reject(new Error(`frame-extract timeout after ${FRAME_EXTRACT_TIMEOUT_MS}ms`)),
            FRAME_EXTRACT_TIMEOUT_MS,
          )
        })
        await Promise.race([exec, timeout])
      } catch (err) {
        // eslint-disable-next-line no-console
        console.warn('[ffmpeg:frames] aborted/failed:', err instanceof Error ? err.message : err)
        return []
      } finally {
        if (timeoutId !== null) clearTimeout(timeoutId)
        ff.off('log', onLog)
      }

      const frames: string[] = []
      for (let i = 1; i <= FRAME_TARGET; i++) {
        const name = `frame_${String(i).padStart(3, '0')}.jpg`
        try {
          const data = (await ff.readFile(name)) as Uint8Array
          frames.push(`data:image/jpeg;base64,${uint8ToBase64(data)}`)
          await ff.deleteFile(name)
        } catch { break }
      }
      return frames
    }
    ```

    ```typescript
      if (
        status.kind !== 'picked' &&
        status.kind !== 'error' &&
        status.kind !== 'done'
      ) return
      const file = status.file
      if (!file) return

      const myGen = ++generationRef.current

      setStatus({ kind: 'analysing', file, step: null, preparing: true })

      try {
        const result = await analyse(file, {
          signal: controller.signal,
          onProgress: (step) => {
            if (myGen !== generationRef.current) return
            setStatus((prev) => {
              if (prev.kind !== 'analysing') return prev
              return { ...prev, step, preparing: false }
            })
    ```
