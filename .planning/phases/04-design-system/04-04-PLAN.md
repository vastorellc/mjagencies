<objective>
Ship the 12 niche default themes as filesystem-only `theme-{slug}.json` files (one per agency in `AGENCIES`), all OKLCH-based per RESEARCH §5.1, all validating against `theme.schema.json` (Plan 04-02), all compilable via `compileThemeToCss` (Plan 04-03). The plan also establishes the single TypeScript source of truth for niche palettes (`niche-palettes.ts`) and font stacks (`font-stacks.ts`) that the 12 JSON files derive from. The build-time generator (`scripts/generate-agency-css.ts`) emits a single combined `agencies.generated.css` consumed by `packages/ui/styles/theme.css` — adding the agency override layer (Layer 2 of the 5-layer cascade) to every Tailwind build.

Purpose: REQ-044 (12 niche themes pre-built), REQ-042 (each theme has 20 scopes — validated), REQ-046 (each theme has dark overlay). Plan 04-05 (Storybook) will toggle `data-agency="X"` on the preview decorator, generating 540 stories (45 blocks × 12 themes — implemented in Phase 5). Phase 8 agency apps will spread `getDataAttrs({ agency: 'ecommerce' })` (Plan 04-03) on `<html>` and the agency CSS block from this plan will activate via cascade.

Plan-time decisions captured (RESEARCH §5, §A1 Q5 Open Q4, §A1 Q1 Q2):
- **OKLCH-only palettes (REQ-047 + perceptual uniformity).** RESEARCH §5.1 lists the 12 niche palettes as OKLCH triples. Plan 04-04 ships exactly those values — no hex anywhere in theme.json. Layer 1 primitives (Plan 04-01) provide the base; per-agency themes reference primitives where palettes share (e.g., neutrals) or define new OKLCH triples for the agency's specific brand color.
- **Single TS source + generated JSON.** `niche-palettes.ts` is the human-edit source. The 12 JSON files are GENERATED — but committed (not gitignored) so editors and the AJV validator see them and Plan 04-05 Storybook can read them statically. The script `scripts/generate-themes.ts` (separate from the CSS generator) is OPTIONAL — at M004 the JSON files can be hand-written from the TS source for clarity. The generation script (`scripts/generate-agency-css.ts`) is REQUIRED because the CSS output is build-derived.
- **Open Q4 resolved (no DB migration here):** Confirmed via Plan 04-03's resolution. Themes are filesystem-only at M004. `agency_themes` and `theme_versions` Drizzle migrations defer to Phase 5 CMS (`005_create_theme_tables.sql`). This plan ships ZERO new SQL.
- **`agencies.generated.css` is GENERATED but committed.** Many monorepos gitignore generated files; here we COMMIT the file so:
  - Plan 04-05 Storybook builds without running a generation step in CI.
  - `web-main` builds without a generation step (until Phase 8 agency apps wire turbo-pipelines).
  - Diff review in PRs surfaces unintended palette changes.
  The script is idempotent — running `pnpm theme:generate` twice produces identical output. CI can add a check (`git diff --exit-code packages/ui/styles/agencies.generated.css`) in Plan 04-05 to ensure the committed version stays in sync.
- **`@import` order in theme.css matters.** `agencies.generated.css` imports between dark-mode (Layer 4 of cascade) and `@theme inline` (Tailwind bridge). This means agency token overrides fire AFTER `:root` base + dark mode, but BEFORE the Tailwind utility resolution. The data-* attribute selector specificity correctly orders runtime cascade.
- **Niche font assignment from RESEARCH §5.2 (Open Q5 partial — `next/font/google` interaction).** Each theme.json `scopes.type.font-brand` is a font-family string that aligns with the `next/font/google` setup Phase 8 will use. The Layer 3 typography token (Plan 04-01) already wires `--mj-font-brand: var(--font-brand, fallback)` — agency apps load `font-brand` via `next/font/google`, the token reads it through fallback indirection. The font NAME in theme.json is the human-readable family (`'Plus Jakarta Sans, system-ui, sans-serif'`) — used both for non-Next-font contexts (e.g., emails — Phase 9) and as the fallback inside the var() chain.
- **Dark overlay scope = COLORS ONLY.** Per Plan 04-01 dark-mode.css pattern — only Layer 2 colors flip. Each theme's `dark.color` block flips bg-primary, bg-secondary, text-primary, text-secondary, border-default. Brand-tone shift may be present (e.g., `accent-primary` shifts to a slightly lighter version for dark backgrounds) but spacing/typography/components do not flip.

```ts
// scripts/generate-agency-css.ts
// Reads 12 theme.json files; emits one combined CSS file with all agency override blocks.
// Run: pnpm theme:generate
import { readFileSync, writeFileSync, readdirSync } from 'node:fs';
import { join } from 'node:path';
import { compileThemeToCss } from '../packages/ui/src/theme/compile-theme.js';
import { assertValidTheme } from '../packages/ui/src/theme/validate-theme.js';
import type { ThemeJson } from '../packages/ui/src/theme/types.js';
import { AGENCIES } from '../packages/config/src/agency-constants.js';

const THEMES_DIR = 'packages/ui/themes/default';
const OUTPUT     = 'packages/ui/styles/agencies.generated.css';

const HEADER = `/* AUTO-GENERATED by scripts/generate-agency-css.ts.
 * DO NOT EDIT MANUALLY. Regenerate via: pnpm theme:generate
 * Source: packages/ui/themes/default/theme-{agency}.json
 * Spec: REQ-044 (12 niche default themes), REQ-046 (dark overlay)
 */
`;

function main(): void {
  const blocks: string[] = [HEADER];
  const onDisk = new Set(
    readdirSync(THEMES_DIR).filter(f => f.startsWith('theme-') && f.endsWith('.json')),
  );
  const expected = new Set(AGENCIES.map(a => `theme-${a}.json`));

  // Sanity check: 12 agencies → 12 files
  for (const exp of expected) {
    if (!onDisk.has(exp)) {
      throw new Error(`Missing theme file: ${exp}. Plan 04-04 must ship all 12.`);
    }
  }
  for (const found of onDisk) {
    if (!expected.has(found)) {
      console.warn(`  Extra file (will be included anyway): ${found}`);
    }
  }

  // Compile each agency theme to CSS
  for (const agency of AGENCIES) {
    const file = join(THEMES_DIR, `theme-${agency}.json`);
    const data = JSON.parse(readFileSync(file, 'utf-8')) as unknown;
    assertValidTheme(data, `theme-${agency}.json`);
    const css = compileThemeToCss({ agency, theme: data as ThemeJson });
    blocks.push(`/* ===== ${agency} ===== */\n${css}\n`);
  }

  writeFileSync(OUTPUT, blocks.join('\n'), 'utf-8');
  console.log(`  Generated ${OUTPUT} from ${AGENCIES.length} agency themes.`);
}

main();
```
