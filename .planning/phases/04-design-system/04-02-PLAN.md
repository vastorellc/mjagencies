<objective>
Build the JSON Schema validator for `theme.json` files and the SVG hex-literal scanner. The schema enforces all 20 customization scopes are present (REQ-042) and rejects hex literals in token values (REQ-047). AJV 8.20.0 compiles the schema once at module init (Pitfall 4) and exports `assertValidTheme()` as a TypeScript assertion function — Plan 04-03 (theme compiler) and Plan 04-04 (12 niche themes) consume this directly. The CI script `scripts/validate-themes.ts` is wired to root `package.json` as `pnpm theme:validate`, and is added to the GitHub Actions pipeline by Plan 04-05.

Purpose: Plan 04-04 ships 12 `theme-{slug}.json` files — every file MUST validate against this schema or it cannot ship. Plan 04-03 (theme compiler) consumes `assertValidTheme()` to fail fast on bad input before generating CSS. The SVG scanner (`assertNoHexLiterals`) enforces REQ-047 against the 30 niche illustrations per agency that ship in Phase 12 content sprint — at M004 only the validator exists; assets arrive later but the gate is live now.

Plan-time decisions captured (RESEARCH §2, §Security Domain):
- **AJV 8.20.0 over Zod or io-ts.** RESEARCH §11 locks `ajv@8.20.0` + `ajv-formats@3.0.1`. Zod cannot express `not.pattern` cleanly in JSON-form schemas, and we need a portable Draft-07 schema that VS Code and editor tooling can validate against directly via the `$schema` reference. AJV is the runtime + the schema is the source of truth.
- **No-hex pattern is `not: { pattern: "#[0-9a-fA-F]{3,8}\\b" }`.** Matches 3-8 hex chars after `#`, terminated by word boundary. Rejects `#fff`, `#ffffff`, `#ffffffff` (rgba hex). Allows `#rrggbb_extension` (e.g., id selectors in custom-css string scope) only because custom-css is `type: string` without the no-hex constraint applied (escape hatch for one-off CSS — RESEARCH §4 scope 19).
- **No-url pattern is `not: { pattern: "url\\(" }` on scope values** (color/brand/etc.) to block `url(data:image/...)` SSRF and `url(javascript:...)` legacy injection. The `imagery` scope and `code-injection` scope are EXCLUDED from this rule because legitimate image URLs may appear there — those scopes use the `string` type without no-url.
- **20 scopes are REQUIRED.** Schema lists all 20 in `required`. `additionalProperties: false` on the top-level scopes object so adding a 21st scope is a deliberate schema change (not a typo bypass).
- **Module singleton.** The compiled validator is created at module load via `const validateTheme = ajv.compile<ThemeJson>(themeSchema)` and reused on every call. AJV docs explicitly recommend this (Pitfall 4 mitigation).
- **TypeScript assertion vs predicate.** `assertValidTheme` uses `asserts data is ThemeJson` syntax so callers don't need to cast — TS narrows automatically. Compare with `isValidTheme` (predicate) which would force callers to write `if (!isValidTheme(d)) throw ...`. Assertion is ergonomically better for the 12 plan-04-04 themes.
- **SVG scanner is regex-only, not XML-parsing.** A real XML parser would correctly handle CDATA / attribute escaping, but adds a dependency. The hex-literal regex `/#[0-9a-fA-F]{3,8}\b/` is deterministic across SVG syntactic variations (attribute values, CSS-in-style blocks, gradient stops). Pitfall 7 confirms this is the standard approach.

Output: `pnpm --filter=@mjagency/ui typecheck` exits 0. `pnpm --filter=@mjagency/ui vitest run` exits 0 with at least 12 validator tests passing. `pnpm theme:validate` runs from repo root, succeeds (or skips cleanly) when no themes/SVGs exist yet — the script must handle empty directories gracefully because Plan 04-04 ships the 12 themes and the 30+ SVGs arrive in Phase 12. CI security-grep continues clean.

// packages/ui/src/theme/validate-theme.ts
import Ajv, { type ValidateFunction } from 'ajv';
import addFormats from 'ajv-formats';
import { readFileSync } from 'node:fs';
import { join, dirname } from 'node:path';
import { fileURLToPath } from 'node:url';
import type { ThemeJson } from './types.js';

// Load schema synchronously at module init — readFileSync is safe here because
// validate-theme.ts is server-only / build-only (never bundled to browser).
const __dirname = dirname(fileURLToPath(import.meta.url));
const SCHEMA_PATH = join(__dirname, '../../themes/schemas/theme.schema.json');
const themeSchema = JSON.parse(readFileSync(SCHEMA_PATH, 'utf-8')) as object;

// AJV singleton — compile once, reuse forever (Pitfall 4)
const ajv = new Ajv({ allErrors: true, strict: true });
addFormats(ajv);
const validateTheme: ValidateFunction<ThemeJson> = ajv.compile<ThemeJson>(themeSchema);

/**
 * TypeScript assertion: narrows `unknown` to `ThemeJson` on success.
 * Throws Error with formatted error list on failure (REQ-041).
 */
export function assertValidTheme(data: unknown, filename: string): asserts data is ThemeJson {
  const valid = validateTheme(data);
  if (!valid) {
    const errors = validateTheme.errors ?? [];
    const messages = errors
      .map(e => `  ${e.instancePath || '(root)'}: ${e.message}${e.params ? ' — ' + JSON.stringify(e.params) : ''}`)
      .join('\n');
    throw new Error(`Invalid theme "${filename}":\n${messages}`);
  }
}

/**
 * Standalone hex-literal scanner — used for SVG illustrations.
 * Belt-and-suspenders: AJV catches hex in theme.json values; this catches hex in SVG XML (REQ-047, Pitfall 7).
 */
const HEX_LITERAL_RE = /#[0-9a-fA-F]{3,8}\b/g;
export function assertNoHexLiterals(content: string, filename: string): void {
  const matches = content.match(HEX_LITERAL_RE);
  if (matches && matches.length > 0) {
    const unique = Array.from(new Set(matches));
    throw new Error(
      `Hex literal(s) detected in "${filename}": ${unique.join(', ')}. ` +
      `Use var(--mj-ill-*) instead. See packages/ui/tokens/layer-6-components.css for available illustration tokens.`
    );
  }
}
```

// scripts/validate-themes.ts
// Runs in CI: pnpm theme:validate
import { readFileSync, readdirSync, existsSync } from 'node:fs';
import { join } from 'node:path';
import { assertValidTheme, assertNoHexLiterals } from '../packages/ui/src/theme/validate-theme.js';

const THEMES_DIR = 'packages/ui/themes/default';
const SVG_DIR    = 'packages/ui/assets/illustrations';

let errorCount = 0;
let okCount    = 0;

// Validate all theme.json files (Plan 04-04 ships 12 of these)
if (existsSync(THEMES_DIR)) {
  const files = readdirSync(THEMES_DIR).filter(f => f.endsWith('.json'));
  for (const file of files) {
    try {
      const data = JSON.parse(readFileSync(join(THEMES_DIR, file), 'utf-8'));
      assertValidTheme(data, file);
      console.log(`  theme OK: ${file}`);
      okCount++;
    } catch (err) {
      console.error(`  theme FAIL: ${file}\n${(err as Error).message}\n`);
      errorCount++;
    }
  }
} else {
  console.log(`  (no themes directory yet at ${THEMES_DIR} — Plan 04-04 ships 12 niche themes)`);
}

// Check all SVG files for hex literals (Phase 12 content sprint ships 30+/agency)
if (existsSync(SVG_DIR)) {
  const files = readdirSync(SVG_DIR).filter(f => f.endsWith('.svg'));
  for (const file of files) {
    try {
      const content = readFileSync(join(SVG_DIR, file), 'utf-8');
      assertNoHexLiterals(content, file);
      console.log(`  svg OK: ${file}`);
      okCount++;
    } catch (err) {
      console.error(`  svg FAIL: ${file}\n${(err as Error).message}\n`);
      errorCount++;
    }
  }
} else {
  console.log(`  (no illustrations directory yet at ${SVG_DIR} — Phase 12 content sprint)`);
}

if (errorCount > 0) {
  console.error(`\n  ${errorCount} validation error(s); ${okCount} OK`);
  process.exit(1);
}
console.log(`\n  ${okCount} item(s) validated. No hex literals found. No invalid themes.`);
```
