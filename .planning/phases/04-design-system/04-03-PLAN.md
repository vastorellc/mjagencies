<objective>
Build the theme resolution stack as a pure CSS cascade — five `[data-*]` attribute layers stacked on `<html>` (`data-agency` → `data-page` → `data-theme` → `data-variant`) with the base `:root` underneath. NO JavaScript object merging at runtime, NO `style.setProperty` loops — `setAttribute` on the root element propagates instantly via CSS variable inheritance. This is the architectural backbone Plan 04-04 (12 niche themes) generates `[data-agency="X"]` blocks for, and Plan 04-05 (Storybook) toggles between via a decorator. The plan also ships the FOUC-prevention inline script and `useTheme()` client hook so dark mode never flashes on first paint.

Purpose: REQ-043 (base → agency → page resolution), REQ-045 (theme switch < 16ms), REQ-046 (dark mode via token swap, no asset reload). Plan 04-04 emits one `[data-agency="X"]` block per agency using `compileThemeToCss()`. Plan 04-05's Storybook decorator sets `data-agency` + `data-theme` globals to render every block in every theme. Phase 8 agency apps consume `getDataAttrs()` in their root layouts. Phase 5 CMS will later persist agency theme JSON to `agency_themes` table — at M004 themes are filesystem-only (Open Q4 resolution: defer DB persistence to M005).

Plan-time decisions captured (RESEARCH §3, §6.2, Open Q4):
- **5 layers, single root anchor.** All `data-*` attributes attach to `<html>` only. RESEARCH §3.1 specifies `data-agency`, `data-page`, `data-theme`, `data-variant` — plus the base `:root`. Five attribute layers; CSS specificity orders them by selector compound count + cascade order.
- **Pure CSS cascade — no JS merging.** RESEARCH §3.1 explicitly rejects JS object merging at runtime. The cascade is the merger. This is what makes <16ms theme switch achievable: `setAttribute` is a single DOM mutation that the browser propagates via the existing styling pipeline.
- **`compileThemeToCss` is build-time, not runtime.** Plan 04-04 will run this once per agency at build to emit static CSS that ships with the app bundle. The runtime cost is zero — no compilation in the request path.
- **`compileThemeToCss` calls `assertValidTheme` first.** Plan 04-02's validator is the gate. If a theme.json is malformed, the build fails; CSS is never generated from invalid input.
- **`custom-css` scope = post-block append, not interpolation.** RESEARCH §3.2 + Pitfall 8: the `custom-css` string is appended VERBATIM after the `[data-agency="..."]` block — it is admin-authored raw CSS. At M004 admin-only (no user input). Phase 10+ Builder must add `expression(`/`javascript:`/`url(data:` strip before this scope is widened — DEFERRED. This plan only ensures the compiler does not silently inject `custom-css` inside the scope block (which would inherit the data-agency selector and pollute downstream cascade).
- **Open Q4 (DB migration timing):** RESEARCH §A1 Q5 asks whether `agency_themes` table needs a Drizzle migration in Phase 4. Resolution: **NO migration in Phase 4.** Themes are filesystem-only at M004 (`packages/ui/themes/default/theme-{slug}.json` shipped by Plan 04-04). DB persistence (`agency_themes`, `theme_versions`, `theme_ab_variants`) defers to Phase 5 CMS — `005_create_theme_tables.sql` will live there. This avoids cross-phase migration sequencing and lets M004 ship without DB dependency.
- **`resolveTheme()` is a thin server-side selector at M004.** It reads filesystem (`packages/ui/themes/default/theme-{agency}.json`), parses, validates, returns the resolution. Phase 5 will add DB-backed override; Phase 9 will add A/B variant lookup. The interface is stable: `resolveTheme({ agencyId, page, variant })` returns `{ theme: ThemeJson, dataAttrs: Record<string, string> }`.
- **FOUC inline script is dangerouslySetInnerHTML — admin-justified.** RESEARCH §6.2 + Pitfall 3: blocking script in `<head>` is the standard pattern. The script content is a static literal string — NOT user input. CSP nonce will be wired in Phase 11 (the script today is unconditional inline; Phase 11 adds the nonce attribute). `suppressHydrationWarning` on `<html>` is mandatory because the script mutates `data-theme` before React hydrates.
- **Playwright e2e test optional (graceful skip).** Per RESEARCH §3.3 the perf test is `setAttribute` → `performance.now()` delta. Plan ships the test file; if Playwright is not yet provisioned in CI runner (it gates on Phase 1 / 1.5 setup), the test skips cleanly via `test.skip()`. The vitest unit tests carry the bulk of the verification.

// packages/ui/src/theme/compile-theme.ts
// Build-time only. Plan 04-04 runs this once per agency at build to emit static CSS.
import { assertValidTheme } from './validate-theme.js';
import type { ThemeJson, ScopeKey } from './types.js';

interface CompileOptions {
  agency: string;        // slug, e.g., 'ecommerce'
  theme:  ThemeJson;
}

const SKIP_DURING_SCOPE_BLOCK: ReadonlySet<ScopeKey> = new Set(['custom-css', 'code-injection']);

/**
 * Emit per-agency CSS:
 *   [data-agency="X"] { --mj-{scope}-{key}: {value}; ... }
 *   [data-agency="X"][data-theme="dark"] { ... }   // optional
 *   {custom-css verbatim}                          // optional, post-block
 *
 * REQ-043. Calls assertValidTheme first — fail fast on invalid input.
 */
export function compileThemeToCss({ agency, theme }: CompileOptions): string {
  assertValidTheme(theme, `theme-${agency}.json`);

  const lines: string[] = [];
  const { scopes, dark } = theme;

  // Layer 2: agency override block
  lines.push(`[data-agency="${agency}"] {`);
  for (const [scope, values] of Object.entries(scopes) as [ScopeKey, unknown][]) {
    if (SKIP_DURING_SCOPE_BLOCK.has(scope)) continue;
    if (typeof values === 'object' && values !== null) {
      for (const [key, value] of Object.entries(values as Record<string, string>)) {
        lines.push(`  --mj-${scope}-${key}: ${value};`);
      }
    }
  }
  lines.push('}');

  // Layer 4: dark mode override block (only if `dark` overlay is provided)
  if (dark && Object.keys(dark).length > 0) {
    lines.push(`[data-agency="${agency}"][data-theme="dark"] {`);
    for (const [scope, values] of Object.entries(dark)) {
      if (typeof values === 'object' && values !== null) {
        for (const [key, value] of Object.entries(values as Record<string, string>)) {
          lines.push(`  --mj-${scope}-${key}: ${value};`);
        }
      }
    }
    lines.push('}');
  }

  // Custom CSS (admin-only escape hatch — appended verbatim, NOT inside scope block)
  // M004: admin-only; M010 Builder must strip expression(/javascript:/url(data: before widening this scope.
  const customCss = scopes['custom-css'];
  if (typeof customCss === 'string' && customCss.trim().length > 0) {
    lines.push(`/* custom-css for ${agency} (admin-authored, no compile-time interpolation) */`);
    lines.push(customCss);
  }

  return lines.join('\n');
}
```

// packages/ui/src/theme/resolve-theme.ts
// M004: filesystem read. M005+: DB-backed via agency_themes table.
import 'server-only';
import { readFileSync, existsSync } from 'node:fs';
import { join } from 'node:path';
import { assertValidTheme } from './validate-theme.js';
import type { ThemeJson } from './types.js';

export interface ThemeResolutionInput {
  agency:   string;
  page?:    string;
  theme?:   'light' | 'dark';
  variant?: 'a' | 'b';
  themesDir?: string;  // override for tests; defaults to packages/ui/themes/default
}

export interface ThemeResolution {
  theme:     ThemeJson;
  dataAttrs: Record<string, string>;
}

const DEFAULT_THEMES_DIR = 'packages/ui/themes/default';

/**
 * Resolve the active theme + Layer-1..5 data-attrs for SSR injection.
 * REQ-043 — base → agency → page; dark + variant attached as orthogonal layers.
 * M004: filesystem read. M005: DB read with filesystem fallback.
 */
export function resolveTheme(input: ThemeResolutionInput): ThemeResolution {
  const themesDir = input.themesDir ?? DEFAULT_THEMES_DIR;
  const file = join(themesDir, `theme-${input.agency}.json`);
  if (!existsSync(file)) {
    throw new Error(`No theme.json for agency "${input.agency}" at ${file}. ` +
                    `Plan 04-04 ships 12 default themes; verify the slug matches AGENCIES const.`);
  }

  const data = JSON.parse(readFileSync(file, 'utf-8'));
  assertValidTheme(data, `theme-${input.agency}.json`);

  const dataAttrs: Record<string, string> = {
    'data-agency':  input.agency,
    'data-theme':   input.theme   ?? 'light',
  };
  if (input.page)    dataAttrs['data-page']    = input.page;
  if (input.variant) dataAttrs['data-variant'] = input.variant;

  return { theme: data, dataAttrs };
}
```

```tsx
// packages/ui/src/hooks/use-theme.ts
'use client';
import { useCallback, useEffect, useState } from 'react';

const STORAGE_KEY = 'mj-theme';

export function useTheme() {
  const [theme, setThemeState] = useState<'light' | 'dark'>('light');

  useEffect(() => {
    // Read DOM-truth (FOUC script set this) + localStorage as fallback
    const stored  = localStorage.getItem(STORAGE_KEY) as 'light' | 'dark' | null;
    const current = document.documentElement.getAttribute('data-theme') as 'light' | 'dark';
    setThemeState(stored ?? current ?? 'light');
  }, []);

  const setTheme = useCallback((next: 'light' | 'dark') => {
    document.documentElement.setAttribute('data-theme', next);  // INSTANT (REQ-045)
    localStorage.setItem(STORAGE_KEY, next);
    setThemeState(next);
  }, []);

  const toggle = useCallback(() => {
    setTheme(theme === 'light' ? 'dark' : 'light');
  }, [theme, setTheme]);

  return { theme, setTheme, toggle } as const;
}
```

'use client';
import { useCallback, useEffect, useState } from 'react';

export function usePageTheme() {
  const [page, setPageState] = useState<string | null>(null);

  useEffect(() => {
    const current = document.documentElement.getAttribute('data-page');
    setPageState(current);
  }, []);

  const setPageTheme = useCallback((next: string | null) => {
    if (next === null) {
      document.documentElement.removeAttribute('data-page');
    } else {
      document.documentElement.setAttribute('data-page', next);  // INSTANT
    }
    setPageState(next);
  }, []);

  return { page, setPageTheme } as const;
}
