<objective>
Build a parallel-by-default, per-agency migration runner that applies the Drizzle schema (Plan 02-01 generated `0000_initial.sql` + the two custom DDL files) across all 13 agency databases. The runner connects directly to Postgres on port 5432 (bypassing PgBouncer per pitfall 8.2) using the `migrations_runner` BYPASSRLS role provisioned in Plan 02-01. It supports `--dry-run`, `--canary` (brand first → confirm → rest), `--sequential` (fallback), `--snapshot-before` (schema dump for rollback), and the default `--all` parallel mode using `Promise.allSettled`.

Purpose: REQ-015 demands "migration runner — parallel, dry-run, canary, rollback". Plan 02-04 (seed framework), Plan 02-05 (backups), and Plan 02-06 (vault + audit triggers) all depend on the schema being applied to the 13 DBs first. This plan is the gate that takes the schema from "designed" to "deployed in dev/CI/prod". Per RESEARCH §2.1, drizzle-kit's CLI cannot fan out across 13 DBs in parallel; the programmatic API `migrate(db, { migrationsFolder })` from `drizzle-orm/postgres-js/migrator` is the correct primitive — one call per agency, all fired concurrently, each with its own dedicated `max=1, prepare=false` postgres-js client.

The runner also handles the **two custom DDL files** that drizzle-kit doesn't generate: `001_agency_id_immutable.sql` (the BEFORE UPDATE trigger function + per-table triggers) and `002_force_rls_and_app_role.sql` (FORCE RLS + REVOKE on audit_log + per-agency role grants). The runner reads each file, performs simple `:'app_role'` → `<slug>_user` regex substitution, and applies via the `migrations_runner` connection. This sequencing is non-trivial enough to warrant explicit unit tests (dry-run output) and integration tests (full apply against a fresh DB) — both included.

Plan-time decisions captured (RESEARCH §A1):
- **Open Q1 resolved:** add `buildDirectUrl(slug, password)` to `packages/db/src/connection.ts`. Returns a postgres URL on port 5432 with the per-agency dbName but the `migrations_runner` role. Used EXCLUSIVELY by the migration runner. Documented in JSDoc with a clear warning: "DO NOT use for application queries — bypasses PgBouncer pool and runs as BYPASSRLS role."
- **Open Q2 resolved:** the runner reads `MIGRATIONS_DB_PASSWORD` (single shared) from env. Documented as a Doppler shared-project secret in `docs/runbooks/migrations.md`. Plan 01-06 (Doppler) is currently deferred but does not block this plan — env var is the integration point regardless.
- **Rollback strategy:** `--snapshot-before` calls `pg_dump --schema-only` per agency before the apply step. On failure, `scripts/migrate-rollback.ts <snapshot-dir>` runs `psql -f <snapshot>` to restore. There is no automatic rollback (Drizzle does not generate down migrations — RESEARCH §2.3).

Output: `pnpm tsx scripts/migrate-runner.ts --dry-run --all` runs against a populated dev stack and prints "0 pending migrations × 13 DBs" or the actual pending list per agency. `pnpm tsx scripts/migrate-runner.ts --canary` migrates `brand` first, prompts, then fans out the rest. Unit tests cover dry-run output formatting; integration tests (skipIf without DB) cover the full apply path against an ephemeral Postgres.
</objective>

Locked direct-URL helper signature (Open Q1):
```ts
// packages/db/src/connection.ts (NEW EXPORT)

/**
 * Builds a DIRECT Postgres URL (port 5432, bypassing PgBouncer) for migration use ONLY.
 *
 * Returns a connection string authenticated as `migrations_runner` (the BYPASSRLS role
 * provisioned in Plan 02-01). NEVER use this for application queries — application code
 * goes through PgBouncer (port 6432-6443) as `<slug>_user`.
 *
 * Direct port 5432 is required because PgBouncer transaction mode breaks multi-statement
 * DDL (pitfall 8.2). The migration runner is the sole legitimate consumer.
 *
 * @example
 *   const url = buildDirectUrl('ecommerce', process.env.MIGRATIONS_DB_PASSWORD!)
 *   // 'postgresql://migrations_runner:***@127.0.0.1:5432/ecommerce_db'
 */
export function buildDirectUrl(
  slug: (typeof AGENCIES)[number],
  password: string
): string {
  const conn = agencyConnection(slug)
  return `postgresql://migrations_runner:${encodeURIComponent(password)}@127.0.0.1:5432/${conn.dbName}`
}
```

```ts
// packages/db/src/migrate/runner.ts (skeleton)
import { migrate } from 'drizzle-orm/postgres-js/migrator'
import { drizzle } from 'drizzle-orm/postgres-js'
import postgres from 'postgres'
import { buildDirectUrl } from '../connection.js'

const MIGRATIONS_FOLDER = './packages/db/src/migrations'

export async function runMigration(slug: string, password: string): Promise<void> {
  const url = buildDirectUrl(slug as any, password)
  const client = postgres(url, { max: 1, prepare: false })  // pitfall 8.3
  const db = drizzle({ client })
  try {
    await migrate(db, { migrationsFolder: MIGRATIONS_FOLDER })
    // After drizzle-kit generated migrations: apply custom DDL
    await applyCustomDdl(client, slug)
  } finally {
    await client.end()
  }
}
```

Locked custom DDL substitution (the two files don't have `:'app_role'`/`:"DB_NAME"` actually populated by Drizzle — runner injects):
```ts
// packages/db/src/migrate/apply-custom.ts (skeleton)
import { readFile } from 'node:fs/promises'
import path from 'node:path'
import type { Sql } from 'postgres'

const CUSTOM_DIR = './packages/db/src/migrations/custom'
const CUSTOM_FILES = [
  '001_agency_id_immutable.sql',
  '002_force_rls_and_app_role.sql',
]

export async function applyCustomDdl(client: Sql, slug: string): Promise<void> {
  for (const file of CUSTOM_FILES) {
    const raw = await readFile(path.join(CUSTOM_DIR, file), 'utf8')
    const substituted = raw
      .replace(/:'app_role'/g, `${slug}_user`)
      .replace(/:"app_role"/g, `${slug}_user`)
    // postgres.js does not run `\connect` or psql meta-commands — strip them
    const sqlBody = substituted.split('\n').filter(l => !l.trimStart().startsWith('\\')).join('\n')
    await client.unsafe(sqlBody)  // multi-statement DDL — port 5432 only (pitfall 8.2)
  }
}
```

```ts
// packages/db/src/migrate/snapshot.ts
import { spawn } from 'node:child_process'
import path from 'node:path'

export async function snapshotAgency(
  slug: string, password: string, outDir: string
): Promise<string> {
  const file = path.join(outDir, `${slug}.sql`)
  // pg_dump runs against direct port 5432 as migrations_runner
  const args = [
    '--host=127.0.0.1', '--port=5432',
    `--username=migrations_runner`,
    '--schema-only', '--no-owner', '--no-privileges',
    `--dbname=${slug}_db`,
    `--file=${file}`,
  ]
  const env = { ...process.env, PGPASSWORD: password }
  await new Promise<void>((resolve, reject) => {
    const p = spawn('pg_dump', args, { env, stdio: 'inherit' })
    p.on('exit', code => code === 0 ? resolve() : reject(new Error(`pg_dump ${slug} exit ${code}`)))
  })
  return file
}
```
