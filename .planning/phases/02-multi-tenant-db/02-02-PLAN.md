<objective>
Phase 1 already shipped 12 PgBouncer per-agency configs (transaction mode, pool_size=20, ports 6432-6443) and PM2 supervision in `ecosystem.config.cjs`. This plan does NOT re-implement that infrastructure. Instead, it executes a focused verification + lock-in pass to make sure the RLS context wrapper from Plan 02-01 (`withAgencyContext`) interacts correctly with the existing PgBouncer transaction-mode pools and to install the compile-time + runtime + documentation safety rails that prevent the cross-tenant leak class (pitfall 8.1).

Purpose: Pitfall 8.1 (stale `app.agency_id` leaking across PgBouncer-multiplexed physical connections) is the single most dangerous integration risk in Phase 2 (RESEARCH §1.3). Phase 1 set up PgBouncer correctly, and Plan 02-01 implemented `withAgencyContext` correctly. What's missing is (a) **proof** the two work together end-to-end under concurrency, (b) a **lint rule** preventing future code from accidentally re-introducing session-scoped `SET app.agency_id`, and (c) a **runbook** so on-call engineers can diagnose suspected cross-tenant leaks without re-reading research notes. This plan delivers all three.

Output: A `verify-pgbouncer-rls.ts` script that runs 50 concurrent agency-scoped queries through PgBouncer's connection pool and asserts zero cross-tenant rows leak; an ESLint rule that fails CI if any TS file contains `SET app.agency_id` (use `set_config(...)` instead); a runbook in `docs/runbooks/pgbouncer-rls.md` with the exact correct/wrong patterns; and a re-verification of the Phase 1 .ini files showing they remain to spec (no edits expected — this is a checked-in audit). The plan is autonomous, completes within ~20% context, and can run in parallel with Plan 02-01 (Wave 1) because it modifies only files outside Plan 02-01's surface.

Plan-time decisions:
- **No PgBouncer config changes expected.** This plan re-runs `gen-pgbouncer-config.sh` and asserts byte-equality with checked-in files; if the generator output drifts, the plan fails verification and the executor must reconcile before proceeding. Treat this as a Phase 1 audit gate.
- **`server_reset_query`:** PgBouncer transaction-mode default is `DISCARD ALL`. Because Plan 02-01 uses `set_config(..., true)` (transaction-local), session state is reverted at COMMIT — `DISCARD ALL` is harmless and safe to leave at default. Documented in `pgbouncer-rls.md`.
- **Lint rule scope:** the rule searches for the exact string `SET app.agency_id` in `.ts`/`.tsx` files under `packages/*/src/**` and `apps/*/src/**`. It does NOT match `set_config('app.agency_id', ...)` (which is the correct pattern). Test cases cover both positive and negative.
</objective>

Locked verification pattern (from RESEARCH §1.3 + pitfall 8.1):
```ts
// scripts/verify-pgbouncer-rls.ts (skeleton)
import { createAgencyDb, withAgencyContext, schema } from '@mjagency/db'

async function fanOut() {
  const agents: Promise<unknown>[] = []
  for (let i = 0; i < 50; i++) {
    const slug = AGENCIES[i % 12]
    const agencyId = AGENCY_IDS[slug]
    const db = createAgencyDb(slug, password(slug))
    agents.push(withAgencyContext(db, agencyId, async (tx) => {
      const rows = await tx.select().from(schema.users)
      // Every row's agency_id MUST equal agencyId — assert per row
      for (const row of rows) {
        if (row.agencyId !== agencyId) throw new Error(`LEAK: tx for ${slug} saw row from ${row.agencyId}`)
      }
    }))
  }
  await Promise.all(agents)
}
```

// packages/db/src/lint/no-session-set.ts
import type { Rule } from 'eslint'
export const noSessionSet: Rule.RuleModule = {
  meta: {
    type: 'problem',
    docs: { description: 'Disallow session-scoped SET app.agency_id (pitfall 8.1)' },
    schema: [],
    messages: {
      sessionSet: 'Use `set_config(\'app.agency_id\', id, true)` (SET LOCAL) inside withAgencyContext — session-scoped SET leaks across PgBouncer pool connections.',
    },
  },
  create(context) {
    return {
      Literal(node) {
        if (typeof node.value !== 'string') return
        if (/SET\s+app\.agency_id/i.test(node.value)) {
          context.report({ node, messageId: 'sessionSet' })
        }
      },
      TemplateElement(node) {
        if (/SET\s+app\.agency_id/i.test(node.value.raw)) {
          context.report({ node, messageId: 'sessionSet' })
        }
      },
    }
  },
}
