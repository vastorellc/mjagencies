<objective>
Wire pgBackRest as the canonical backup tool for the shared Postgres 17 instance (which holds all 13 agency DBs per Phase 1 D-04). pgBackRest provides block-level incremental backups, native S3-compatible support (configured for Cloudflare R2), AES-256-CBC encryption at rest, and zstd compression. The plan ships an idempotent config generator (`gen-pgbackrest-config.sh`), three cron-driven backup tiers (daily incremental / weekly differential / monthly full), continuous WAL archiving via `archive_command`, an operator restore tool with PITR support, an R2 smoke test using the existing `@mjagency/media` R2 client, a quarterly DR drill script, and two operator runbooks (daily ops + DR drill).

Purpose: REQ-017 demands "Backup — hourly WAL + snapshots, R2 upload, quarterly DR". Plan 02-06 ships the audit log + permissions vault — those tables are the highest-value backup targets and need the WAL stream catching every change. Phase 12's launch checklist requires a successful DR drill before going live (PROJECT.md SLA: RPO 1h, RTO 4h). This plan's deliverables make those numbers achievable.

Tool selection (RESEARCH §4.1): pgBackRest over barman — block-level incrementals, native S3/R2, single-server simplicity, built-in encryption. barman is enterprise-complexity overkill for the single-VPS architecture from CONTEXT D-01.

Plan-time decisions captured (RESEARCH §A1):
- **Open Q3 resolved:** Single R2 bucket `mjagency-backups` with path prefix per stanza (`repo1-path=/postgres-main/`). Cheaper, simpler IAM, easier lifecycle policies. Per-agency separation already exists at the DB layer (per-DB grants); backup-layer separation is unnecessary at this scale.
- **Cipher choice:** AES-256-CBC via pgBackRest's `repo1-cipher-type=aes-256-cbc`. (Note: pgBackRest does not support AES-GCM at the repo level — Plan 02-06 uses AES-GCM at the **application layer** for the permissions_vault, which is independent. The pgBackRest repo encryption is a transport/at-rest layer for backup files.) Cipher pass sourced from Doppler `PGBACKREST_CIPHER_PASS`.
- **Retention bands** (RESEARCH §4.3, with the 7-year monthly full to match REQ-018 vault retention):
  - WAL: 1095 days (3 years) — sufficient for any business-needed PITR
  - Daily incremental: 30 days
  - Weekly differential: 90 days
  - Monthly full: 7 years (84 months)
- **Cron host:** the same VPS that runs Postgres 17. Cron entries live in `infra/pgbackrest/cron/crontab` (committed); operator installs via `crontab infra/pgbackrest/cron/crontab` at deploy time.
- **Validation strategy:** the R2 smoke test (`pgbackrest-verify-r2.ts`) uses `@mjagency/media`'s `createR2Client` — this is a deliberate cross-package usage. A direct pgbackrest binary call (no client library) handles the actual backup; the smoke test only validates that the bucket is reachable and contains at least one object so we know the upload pipeline is wired correctly.

Output: `bash scripts/gen-pgbackrest-config.sh` produces a deterministic `pgbackrest.conf` consumable by `pgbackrest --config=infra/pgbackrest/pgbackrest.conf backup`. `pnpm tsx scripts/pgbackrest-verify-r2.ts` connects to R2 and reports bucket reachability + object count. `bash scripts/pgbackrest-dr-drill.sh` runs the quarterly drill end-to-end against a staging Postgres. Two committed runbooks document daily ops + DR drill.

This plan does NOT attempt to install pgBackRest on a host (that's a deploy-time/infra task — gated by Phase 12 launch readiness). It ships the configs, scripts, and docs needed to apply them when the host is provisioned.
</objective>

       USAGE="Usage: pgbackrest-restore.sh [--target=<TIMESTAMP>] [--stanza=<NAME>] [--delta]
         --target  ISO 8601 timestamp for PITR, e.g. '2026-04-26 14:30:00'. If omitted, restores latest.
         --stanza  pgBackRest stanza name, default 'postgres-main'
         --delta   Use --delta restore (faster — replaces only diff files)
         --help    Print this help
       "
       STANZA="postgres-main"
       TARGET=""
       DELTA=""
       for arg in "$@"; do
         case "$arg" in
           --target=*) TARGET="${arg#*=}" ;;
           --stanza=*) STANZA="${arg#*=}" ;;
           --delta) DELTA="--delta" ;;
           --help) echo "$USAGE"; exit 0 ;;
           *) echo "Unknown arg: $arg"; echo "$USAGE"; exit 1 ;;
         esac
       done
       echo "[restore] Starting restore for stanza=$STANZA target='$TARGET' delta=$DELTA"
       if [ -n "$TARGET" ]; then
         pgbackrest --stanza="$STANZA" --config=/etc/pgbackrest/pgbackrest.conf restore --target="$TARGET" --target-action=promote $DELTA
       else
         pgbackrest --stanza="$STANZA" --config=/etc/pgbackrest/pgbackrest.conf restore $DELTA
       fi
       echo "[restore] Done. Postgres data dir restored — start the Postgres service to bring online."
