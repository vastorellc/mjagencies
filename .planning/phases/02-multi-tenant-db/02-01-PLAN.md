<objective>
Define the canonical Drizzle schema for the multi-tenant data layer in `packages/db/src/schema/*` — agencies, users, sessions, permissions_vault (skeleton), audit_log (skeleton), and the `_seed_state` resumability table consumed by Plan 02-04. Every agency-scoped table carries an immutable `agency_id` UUID and a Row-Level Security policy that reads the transaction-local `app.agency_id` setting via `current_setting('app.agency_id', true)::uuid` (RESEARCH §1.2).

Purpose: Phase 2 plans 02-03 (migration runner), 02-04 (seed framework), 02-05 (backups), and 02-06 (vault + audit triggers) all depend on this schema being the single source of truth. The `withAgencyContext()` helper exported from `packages/db/src/client.ts` is the ONLY approved query path under PgBouncer transaction mode (pitfall 8.1) — every downstream consumer (apps, scripts, workers) must use it.

This plan also lays the safety rails the rest of the phase relies on: (a) the `prevent_agency_id_change()` PL/pgSQL trigger that enforces REQ-014; (b) `FORCE ROW LEVEL SECURITY` on every tenant table so the table owner cannot silently bypass RLS during integration tests (pitfall 8.4); (c) a `migrations_runner` role with `BYPASSRLS` distinct from per-agency `<slug>_user` roles so application code can never DDL nor bypass RLS by accident; and (d) integration tests proving cross-agency isolation (REQ-013) and `agency_id` immutability (REQ-014) hold under realistic PgBouncer-routed traffic.

Output: `pnpm --filter=@mjagency/db vitest run` exits 0 with at least three integration tests green (schema typing, cross-agency RLS isolation, agency_id immutability). `drizzle-kit generate` produces a committed `0000_initial.sql` consumed by Plan 02-03's runner. `infra/postgres/init.sql.tmpl` is regenerated by `gen-postgres-init.sh` to include the `migrations_runner` role grant per DB. The two custom DDL files (`001_agency_id_immutable.sql` and `002_force_rls_and_app_role.sql`) are sequenced after the generated migration in the runner's apply order.

Plan-time decisions captured (RESEARCH §A1):
- **Open Q5 (RLS + Payload):** RLS in this plan applies ONLY to Phase-2-introduced tables. Payload CMS collections are managed by `withPayload()` and do not get touched here — Phase 5 owns deciding RLS strategy on Payload tables. Documented in `packages/db/README.md`.
- **`FORCE ROW LEVEL SECURITY`:** applied to every Phase-2 agency-scoped table (pitfall 8.4). Migrations connect as `migrations_runner` (BYPASSRLS); app code connects as `<slug>_user` (RLS enforced).
- **Audit log RLS:** disabled on `audit_log` (pitfall 8.8) — append-only via `REVOKE UPDATE, DELETE FROM <slug>_user`. Plan 02-06 wires the row triggers.

Locked client + transaction wrapper (RESEARCH §1.3):
```ts
// packages/db/src/client.ts
import { drizzle } from 'drizzle-orm/postgres-js'
import postgres from 'postgres'
import { sql } from 'drizzle-orm'
import { buildDatabaseUrl } from './connection.js'
import type { AGENCIES } from '@mjagency/config'

export function createAgencyDb(slug: (typeof AGENCIES)[number], password: string) {
  const url = buildDatabaseUrl(slug, password)
  const client = postgres(url, {
    prepare: false,   // pitfall 3.3 carry-over — PgBouncer tx mode
    max: 10,
  })
  return drizzle({ client })
}

export type AgencyDb = ReturnType<typeof createAgencyDb>

export async function withAgencyContext<T>(
  db: AgencyDb,
  agencyId: string,
  callback: (tx: Parameters<Parameters<typeof db.transaction>[0]>[0]) => Promise<T>
): Promise<T> {
  return db.transaction(async (tx) => {
    await tx.execute(sql`SELECT set_config('app.agency_id', ${agencyId}, true)`)
    return callback(tx)
  })
}
```

Locked agency_id immutability trigger (RESEARCH §1.4):
```sql
-- 001_agency_id_immutable.sql
CREATE OR REPLACE FUNCTION prevent_agency_id_change()
RETURNS trigger LANGUAGE plpgsql AS $$
BEGIN
  IF NEW.agency_id IS DISTINCT FROM OLD.agency_id THEN
    RAISE EXCEPTION
      'agency_id is immutable. Attempted change from % to % on table %',
      OLD.agency_id, NEW.agency_id, TG_TABLE_NAME
      USING ERRCODE = 'integrity_constraint_violation';
  END IF;
  RETURN NEW;
END;
$$;
```
