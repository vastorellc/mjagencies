# Phase 6 — UI Design Contract: SEO/AIO/GEO Plugin Engine

> Visual and interaction contract for Payload CMS admin UI components in Phase 6.
> Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> Scope: Payload admin backoffice only. This phase adds zero public-facing UI.
> Three surfaces: (1) SeoPanel sidebar in the Lexical editor, (2) algo_alerts
> dashboard collection view, (3) seo_plugins config panel in the settings collection.

---

## Design System

| Property | Value | Source |
|----------|-------|--------|
| Tool | none (custom var(--mj-*) tokens) | CONTEXT.md + codebase |
| Preset | not applicable | codebase — no components.json |
| Component library | none — inline style objects with --mj-* vars | Existing SeoPanel.tsx + AiPanel.tsx pattern |
| Icon library | none — text indicators and Unicode symbols only in admin panels | Existing admin component pattern |
| Font | var(--mj-font-sans) — system-ui fallback chain via layer-3-typography.css | packages/ui/tokens/layer-3-typography.css |

**Note on shadcn:** This is a Payload CMS admin context, not a Next.js App Router page. Payload renders its own admin shell; custom components are React components injected into Payload's `afterDocControls` / collection view slots. shadcn is not applicable — Payload admin does not use Tailwind class names for injected components. All styling uses inline style objects referencing var(--mj-*) tokens, which Payload admin inherits via the globals.css import chain.

## Spacing Scale

All values reference `packages/ui/tokens/layer-4-layout.css` tokens directly. Components must not hard-code pixel values — use the CSS variable with pixel fallback (e.g. `var(--mj-space-2, 8px)`).

| Token | CSS Variable | Value | Usage in Phase 6 |
|-------|-------------|-------|-------------------|
| xs | --mj-space-1 | 4px | Gap between inline elements (score label / score number), icon-to-text gap |
| sm | --mj-space-2 | 8px | Margin between ScoreBar rows, row gap in algo_alerts list, button padding-y, char counter margin-top |
| base | --mj-space-4 | 16px | Panel padding (all four sides), field label-to-input gap, section heading margin-bottom |
| lg | --mj-space-6 | 24px | Gap between SeoPanel sections (scores section / TL;DR section) |
| xl | --mj-space-8 | 32px | Not used at panel level — reserved for page-level layout |
| 2xl | --mj-space-12 | 48px | Not used in admin panels |

Exceptions:
- Progress bar track height: 4px (not on the 4px grid — this is a fixed UI element dimension matching the existing SeoPanel.tsx pattern, not a spacing value).
- Progress bar margin-top between track and label row: 2px (existing pattern from SeoPanel.tsx — preserve for visual continuity).
- Publish threshold tick mark width: 2px at the threshold % position on the progress bar track (new in Phase 6 — not a spacing value, it is a decorative indicator).
- Touch targets for Regenerate button and Dismiss action: minimum 32px height (admin context, mouse-only UI — no 44px mobile exception needed).

## Typography

All roles use `var(--mj-font-sans)` (system-ui fallback). No custom fonts in admin panels — Payload admin inherits the host page font stack.

| Role | CSS Token | Value | Weight Token | Weight Value | Line Height Token | Line Height | Usage |
|------|-----------|-------|-------------|-------------|-------------------|-------------|-------|
| Panel heading | --mj-text-size-base | 14px | --mj-weight-semibold | 600 | --mj-leading-tight | 1.25 | "SEO / AIO / GEO" section title, "Algorithm Alerts" panel title, "Plugin Configuration" title |
| Label / key | --mj-text-size-sm | 14px | --mj-weight-normal | 400 | --mj-leading-normal | 1.5 | Score bar labels (seo-classic / aio-citations / geo-chunking), field labels in settings config panel |
| Score number | --mj-text-size-sm | 14px | --mj-weight-semibold | 600 | --mj-leading-normal | 1.5 | Numeric score value displayed right-aligned next to each score bar label |
| Supporting / meta | --mj-text-size-xs | 12px | --mj-weight-normal | 400 | --mj-leading-normal | 1.5 | Word count, internal link count, alt coverage, char counter, "updated just now" timestamp, matched keywords list |
| Button label | --mj-text-size-xs | 12px | --mj-weight-semibold | 600 | --mj-leading-tight | 1.25 | "Regenerate" button, "Dismiss" action, "Reset to defaults" action |
| TL;DR textarea | --mj-text-size-sm | 14px | --mj-weight-normal | 400 | --mj-leading-normal | 1.5 | The aio_tldr editable field in SeoPanel |
| Aggregate score | --mj-text-size-xl | 20px | --mj-weight-semibold | 600 | --mj-leading-tight | 1.25 | Overall aggregate score displayed prominently at the top of SeoPanel |

Typography rule: exactly 4 sizes in use across all Phase 6 admin UI — 12px (xs), 14px (sm), 14px (base, same rendered size used at semibold weight for headings), 20px (xl for aggregate). Collapse to 3 effective visual sizes: 12px, 14px, 20px. No 16px or larger type in admin sidebar panels.

**Note on 12px (xs):** The 12px size is intentional for supporting/meta text in the admin panel context (char counter, timestamps, matched keywords, finding list items). Do not collapse to 14px during execution. These elements are secondary information that should recede visually; 12px is the correct size for this density level.

## Color

All values are CSS variable references — no hex literals anywhere in Phase 6 components. Fallback hex values in inline style strings are permitted only as browser fallbacks in the `var(--token, #fallback)` pattern (matching existing SeoPanel.tsx).

| Role | CSS Token | Fallback | Usage |
|------|-----------|----------|-------|
| Dominant (60%) — panel background | --mj-color-bg-primary | #ffffff | SeoPanel panel body, settings config panel body |
| Secondary (30%) — panel border + track | --mj-color-border-default | #e5e7eb | ScoreBar track background, panel border, textarea border, row separator in algo_alerts list |
| Accent (10%) — reserved list below | --mj-color-brand-500 | (per agency) | See accent reserved list |
| Success state | --mj-color-success | #22c55e | ScoreBar fill when score >= 70, "Threshold met" indicator tick |
| Warning state | --mj-color-warning | #f59e0b | ScoreBar fill when score 40–69, "Near threshold" state |
| Error / destructive | --mj-color-error | #ef4444 | ScoreBar fill when score < 40, "Below threshold" indicator, Dismiss button destructive variant, validation error inline message |
| Text primary | --mj-color-text-primary | #111827 | Panel headings, TL;DR textarea value, settings input values |
| Text secondary | --mj-color-text-secondary | #6b7280 | Score bar labels, char counter, supporting meta text, empty state body |
| Focus ring | --mj-color-border-focus | (brand 500) | TL;DR textarea focus outline, Regenerate button focus ring |
| Surface secondary | --mj-color-bg-secondary | #f9fafb | Regenerate button background, Dismiss button background, row hover background in algo_alerts |

**Accent (--mj-color-brand-500) reserved for:**
- Aggregate score donut/ring or number (if score >= publish threshold)
- "Regenerate" button border (not fill — fill is --mj-color-bg-secondary)
- Active/selected row highlight in the algo_alerts list (left border accent)
- Focus outline on all interactive elements (via --mj-color-border-focus)

Accent is NOT used for: score bars (those use success/warning/error state colors), body text, panel backgrounds, or metadata.

**60/30/10 split for each surface:**
- SeoPanel: 60% --mj-color-bg-primary (panel body), 30% --mj-color-border-default (track/border), 10% state colors (score fills + accent CTA)
- algo_alerts list: 60% --mj-color-bg-primary (row background), 30% --mj-color-bg-secondary (alternate rows / hover), 10% --mj-color-error (unreviewed badge) + --mj-color-brand-500 (active row left accent)
- Settings config panel: 60% --mj-color-bg-primary, 30% --mj-color-border-default (field borders), 10% --mj-color-brand-500 (save/apply CTA button)

## Component Inventory

### Component 1: ScoreBar (upgraded)

**File:** `apps/web-main/src/app/(payload)/admin/components/SeoPanel.tsx` (inline component, not a separate file)

**Props:**
```typescript
interface ScoreBarProps {
  label: string           // e.g. "seo-classic"
  score: number           // 0–100
  threshold: number       // publish threshold from config (e.g. 70)
  isExpanded?: boolean    // controls sub-score breakdown visibility
  onToggle?: () => void   // expand/collapse handler
  findings?: string[]     // array of finding strings for expanded view
}
```

**Visual states:**
- Score >= threshold: fill = --mj-color-success; label text-color = --mj-color-text-secondary
- Score >= 40 and < threshold: fill = --mj-color-warning; label text-color = --mj-color-text-secondary
- Score < 40: fill = --mj-color-error; label text-color = --mj-color-text-secondary
- Track has a thin vertical tick mark at the threshold percentage: width 2px, color = --mj-color-text-secondary at opacity 0.4, positioned with `left: {threshold}%`

**Expand/collapse:**
- A chevron icon (Unicode ▾ / ▸ — no icon library dependency) appears right of the score number
- Clicking the row toggles `isExpanded`
- Expanded state: a findings list renders below the track; each finding is a `<li>` at 12px / --mj-color-text-secondary
- Transition: `max-height` from 0 to auto via `overflow: hidden` — no JS animation library

**Interaction contract:**
- Row is `role="button" tabIndex={0}` for keyboard accessibility
- `onKeyDown` handles Enter and Space to toggle
- No hover color change on the bar itself — only cursor changes to `pointer`

### Component 2: AggregateScoreDisplay

**Location:** Top of SeoPanel, above the three ScoreBars

**Visual:**
- Large numeric score (20px / semibold) center or left-aligned
- Label below: "Overall SEO Score" at 12px / --mj-color-text-secondary
- Color of score number follows same threshold logic as ScoreBar (success / warning / error)
- No donut chart — numeric only, for simplicity and zero external dependency

**States:**
- Loading (debounce in progress): score replaced with "—" at --mj-color-text-secondary, no color state
- Error (server action failed): score replaced with "!" + tooltip on hover ("Score unavailable — check editor content") at --mj-color-error

### Component 3: TldrField

**Location:** Second section of SeoPanel, below the three ScoreBars. Separated by a `<hr>` with border = --mj-color-border-default.

**Elements:**
1. Section heading: "AIO Summary (TL;DR)" — 14px / semibold / --mj-color-text-primary
2. Textarea: 3 rows visible, resizable vertically only (`resize: vertical`), `maxLength={120}`, placeholder = "One-sentence summary for AI answer engines. 120 chars max.", font = 14px / normal / --mj-color-text-primary, border = --mj-color-border-default, border-radius = --mj-radius-md, padding = --mj-space-2 (all sides), focus border = --mj-color-border-focus (2px solid)
3. Character counter: displays "{n}/120" at 12px / --mj-color-text-secondary; when n > 110, color switches to --mj-color-warning; when n > 120 (programmatically blocked at 120 but can display on load if field has stale data), color switches to --mj-color-error
4. Regenerate button: to the right of the char counter on the same row; label = "Regenerate"; 12px / semibold; height 28px; padding = var(--mj-space-1, 4px) var(--mj-space-2, 8px); background = --mj-color-bg-secondary; border = 1px solid --mj-color-brand-500; border-radius = --mj-radius-sm; color = --mj-color-brand-500; cursor = pointer; disabled state: opacity 0.5, cursor = not-allowed

**Interaction contract:**
- Textarea `onChange` updates a local state variable; debounce is NOT applied here (this is direct user input, not a trigger for server calls)
- Textarea `onBlur` dispatches Payload `dispatch` to update `aio_tldr` field value (follows Payload form field pattern for custom UI components interacting with form state)
- Regenerate button click fires `generateTldr(agencyId, content)` server action; sets button to loading state (label = "Generating…", disabled = true, opacity 0.5) until server action resolves
- On server action success: textarea value updates; char counter updates; Payload form field value dispatched
- On server action error: inline error message below textarea: "Unable to generate summary. Edit manually." at 12px / --mj-color-error
- Auto-generate on editor open ONLY when `aio_tldr` field value is blank (per CONTEXT.md D-05 clarification in RESEARCH.md Pitfall 5)

### Component 4: AlgoAlertsListView

**Location:** Payload admin collection view for `algo_alerts` collection. Registered as a custom Payload list view component (super_admin only — access control enforced at collection level, not UI level).

**List row columns:**
| Column | Width | Content |
|--------|-------|---------|
| Title | flex-grow | Article title from RSS item, 14px / normal, truncated at 1 line with ellipsis |
| Source | 120px | "Google Search Central" or configurable feed name, 12px / --mj-color-text-secondary |
| Matched keywords | 160px | Comma-separated keyword list, 12px / --mj-color-text-secondary, truncated at 1 line |
| Date | 100px | Relative date ("3 days ago") with ISO date in `title` attribute for hover, 12px / --mj-color-text-secondary |
| Link | 60px | "View" — opens original URL in new tab, 12px / --mj-color-brand-500 |
| Action | 80px | "Dismiss" button |

**Row states:**
- New / unreviewed (`status: 'new'`): left border 3px solid --mj-color-error; background = --mj-color-bg-primary
- Reviewed (`status: 'reviewed'`): left border 3px solid --mj-color-border-default; background = --mj-color-bg-secondary (muted); opacity 0.7

**Dismiss action:**
- Inline "Dismiss" button (28px height, 12px label, --mj-color-bg-secondary background, --mj-color-error border and text color)
- Click fires `setStatus('reviewed')` server action on the record
- No confirmation dialog — this is a low-stakes archival action, not destructive
- Row updates inline on success (optimistic UI: immediately changes row to reviewed style before server confirmation)
- On error: row reverts and shows "Dismiss failed" tooltip on button for 3 seconds

**Empty state:**
- Heading: "No algorithm alerts" at 14px / semibold / --mj-color-text-primary
- Body: "Algorithm monitoring checks every 6 hours. New alerts appear here when Google Search Central or configured RSS feeds match your keyword list." at 12px / --mj-color-text-secondary

**Filter / search:**
- Payload's built-in list view search applies; no custom filter UI needed in Phase 6
- Super_admin can filter by `status` using Payload's native filter panel

### Component 5: SeoPluginsConfigPanel

**Location:** Injected into the Payload `settings` collection edit view as an afterFields component (or custom field UI). Renders below the existing settings fields.

**Structure:**
- Section heading: "SEO Plugin Configuration" — 14px / semibold / --mj-color-text-primary; margin-bottom = var(--mj-space-4, 16px)
- Four sub-sections (one per tunable category from CONTEXT.md D-04): `seo_classic`, `aio_citations`, `geo_chunking`, `score_thresholds`
- Each sub-section uses a `<details>` element for collapsibility (no JS — native HTML disclosure widget)
- `<summary>` label: sub-section name at 14px / semibold, with a "(default)" badge in 12px / --mj-color-text-secondary when no override exists for this agency
- Per-agency override diff display: when an agency has overridden a value, display the field row with two values: "Default: {n}" in 12px / --mj-color-text-secondary and the override input inline

**Input types per sub-section:**

`seo_classic`:
- Title min/max chars: two number inputs (inline, side-by-side) labeled "Title length: min" and "max". Width 64px each.
- Meta desc min/max chars: same pattern
- Keyword density min/max: number inputs with 0.001 step, labeled "Keyword density: min %" and "max %"
- Word count floor: single number input, labeled "Word count floor"
- Internal link minimum: single number input, labeled "Min internal links"

`aio_citations`:
- Required source types: checkboxes for "Government (.gov)", "Academic (.edu)", "News (news.google.com)"

`geo_chunking`:
- Target radius: number input labeled "Service radius (miles)"
- City/region chunk count: number input labeled "Min city mentions per page"
- Required on service pages: checkbox labeled "Require geo content on service pages"

`score_thresholds`:
- Three number inputs (0–100 range): "seo-classic minimum", "aio-citations minimum", "geo-chunking minimum"

**All inputs:** font-size = 14px (--mj-text-size-sm), border = 1px solid --mj-color-border-default, border-radius = --mj-radius-md, padding = var(--mj-space-2, 8px) var(--mj-space-4, 16px), background = --mj-color-bg-primary

**Reset to defaults action:**
- Button at bottom of config panel: "Reset to agency defaults"
- 14px / semibold; background = --mj-color-bg-primary; border = 1px solid --mj-color-error; color = --mj-color-error; border-radius = --mj-radius-md; padding = --mj-space-2 --mj-space-4
- Click shows an inline confirmation row: "This removes all agency overrides and uses platform defaults. Confirm?" with two inline buttons: "Yes, reset" (--mj-color-error filled) and "Keep overrides" (--mj-color-bg-secondary with border)
- No modal/dialog — inline confirmation below the button

## Interaction Contracts

### Real-time Score Updates (SeoPanel)

- Debounce delay: 500ms (mandatory — prevents keystroke-per-request storm per RESEARCH.md Pitfall 2)
- Trigger: `useAllFormFields` `useEffect` dependency — fires on any field change
- Loading state: aggregate score displays "—" and ScoreBars show their last known score (not a spinner) during debounce window and server action in-flight
- Success state: scores update smoothly via React state; `transition: width 0.3s var(--mj-ease-default)` on ScoreBar fill width (existing pattern from SeoPanel.tsx)
- Error state: aggregate shows "!" in --mj-color-error; ScoreBars retain last known score values with a subtle opacity 0.5 applied to all three bars
- Score precision: integer only (Math.round) — no decimal display

### Publish Threshold Indicator

- Each ScoreBar shows the publish threshold as a vertical tick mark on the track
- Tick mark position: `left: {threshold}%` within the track container (absolute positioned)
- Tick mark style: width 2px, height 8px (extends 2px above and 2px below the 4px track), background = --mj-color-text-secondary at opacity 0.35
- No tooltip or label on the tick — the numeric threshold is surfaced in the expanded findings list

### Keyboard Navigation

- All clickable rows and buttons in admin panels must be `tabIndex={0}` with `role="button"` where not a native `<button>`
- Enter and Space activate buttons
- Escape collapses any open expansion in ScoreBar if focused
- Tab order within SeoPanel: aggregate score (display only, skip tab) → ScoreBar rows (tabbable for expand) → TL;DR section heading (skip) → textarea → Regenerate button

### Accessibility

- Color is never the sole indicator of state: score bars show both color fill AND numeric score value
- All `<input>` and `<textarea>` elements have associated `<label>` elements (explicit `htmlFor` / `id` pairing)
- `aria-live="polite"` on the aggregate score container — screen readers announce score changes after debounce settles
- `aria-busy="true"` on SeoPanel root during server action in-flight
- Progress bar tracks use `role="progressbar"` with `aria-valuenow`, `aria-valuemin="0"`, `aria-valuemax="100"`

## Token Usage Rules (Executor Reference)

1. Never use hex literals — use `var(--mj-token, #fallback)` pattern for all color values (matching existing SeoPanel.tsx pattern)
2. Never use pixel values for spacing — use `var(--mj-space-N, Npx)` with fallback
3. Never use px for font-size — use `var(--mj-text-size-*, Npx)` with fallback
4. Never use raw `#` color codes for state colors — success = `var(--mj-color-success, #22c55e)`, warning = `var(--mj-color-warning, #f59e0b)`, error = `var(--mj-color-error, #ef4444)`
5. Border radius: always from --mj-radius-* tokens; use --mj-radius-sm (2px) for buttons, --mj-radius-md (6px) for panel containers and inputs, --mj-radius-full (9999px) for progress bar fills and tracks
6. Transition: always use token-based duration + easing (var(--mj-duration-slow) var(--mj-ease-default) for score bars)
7. Shadow: not used in admin panel components (Payload admin provides its own surface elevation; no extra shadow needed)
