Lock the **server-action auth pattern** into the codebase as a runtime helper + ESLint rule + runbook. The runtime helper is `requireSession()` from `@mjagency/auth` — it reads the access cookie, verifies via `jose` with explicit `algorithms`/`issuer`/`audience`, enforces MFA for super_admin + admin (REQ-024), and redirects to `/login` (or `/mfa/verify`) on failure. The ESLint rule `require-session-first` flags any exported async function in a `'use server'` file whose first statement is not `const session = await requireSession()` (or a parameterized variant). Registered as ERROR severity in `packages/config/eslint/index.js` so all 12 apps inherit it via `eslint.config.js`.

Purpose: REQ-031 / REQ-301 / CLAUDE.md §3 — every server action must verify the session as its FIRST LINE because middleware alone is insufficient (CVE-2025-29927 demonstrated middleware bypass). Plan 03-04's middleware provides fast-path optimistic redirects; this plan provides the strong post-CVE check that runs on the Node side. REQ-024 — MFA enforcement for privileged roles is gated here (the helper redirects to `/mfa/verify` if the token lacks `mfaVerifiedAt`). REQ-310 — `verifyAccessToken` already enforces the locked claims; this plan ensures it's the only path that reaches user data.

Plan-time decisions captured (RESEARCH §7):
- **Helper location:** `requireSession()` lives in `packages/auth/src/require-session.ts` — Node-only (uses `next/headers` `cookies()` and `next/navigation` `redirect()`). NOT exported from `@mjagency/auth/middleware` (that's the lean Edge entry).
- **MFA enforcement:** Auto-detect by role — `super_admin` and `admin` always require MFA. Editor + future roles default to no MFA. Caller can opt-in explicitly via `requireSession({ requireMfa: true })` to force MFA on a specific action regardless of role.
- **ESLint rule scope:** The rule applies to ALL files matching `**/*.ts` and `**/*.tsx` that contain a top-level `'use server'` directive. Files marked with the function-level `'use server'` directive (inside an async function) are NOT covered at this plan — those are rare in practice and a future hardening can extend. Exempt names start with `_` (private helpers).

Locked requireSession helper (RESEARCH §7.1 verbatim):
```ts
// packages/auth/src/require-session.ts
import 'server-only'
import { redirect } from 'next/navigation'
import { verifyAccessToken, type VerifiedAccessPayload } from './tokens.js'
import { readAccessCookie, clearAuthCookies } from './cookie.js'

const MFA_REQUIRED_ROLES = new Set(['super_admin', 'admin'])

export interface RequireSessionOpts {
  /** Force MFA verification for this call regardless of role. Default: false (auto-detect by role). */
  requireMfa?: boolean
}

/**
 * MUST be called as the FIRST statement of every server action (CLAUDE.md §3, REQ-031).
 *
 * Usage:
 *   'use server'
 *   export async function updatePage(data: PageData) {
 *     const session = await requireSession()           // ← FIRST LINE
 *     if (session.agencyId !== data.agencyId) throw new ForbiddenError()
 *     // ... action body
 *   }
 *
 * Throws via Next.js `redirect()` (which itself throws internally) when:
 * - No access cookie is present
 * - JWT verification fails (expired, invalid signature, alg/iss/aud mismatch)
 * - MFA is required (role-based or explicitly requested) but `mfaVerifiedAt` is absent
 *
 * On verify failure, stale cookies are cleared BEFORE redirecting to prevent redirect loops.
 */
export async function requireSession(opts: RequireSessionOpts = {}): Promise<VerifiedAccessPayload> {
  const token = await readAccessCookie()
  if (!token) {
    redirect('/login')  // throws — never returns
  }

  let payload: VerifiedAccessPayload
  try {
    payload = await verifyAccessToken(token)
  } catch {
    await clearAuthCookies()
    redirect('/login')
  }

  // MFA enforcement — auto-detected by role OR explicitly requested
  const mustHaveMfa = opts.requireMfa === true || MFA_REQUIRED_ROLES.has(payload.role)
  if (mustHaveMfa && !payload.mfaVerifiedAt) {
    redirect('/mfa/verify')
  }

  return payload
}
```

```js
// packages/auth/eslint/require-session-first.js
/**
 * ESLint rule: require-session-first
 *
 * In any file with a top-level 'use server' directive, every exported async function
 * MUST have `const ... = await requireSession()` (or `await requireSession({...})`)
 * as its first statement.
 *
 * Exempt: function names starting with '_' (private helpers).
 *
 * Reference: CLAUDE.md §3, REQ-031, docs/runbooks/server-action-pattern.md
 */
'use strict'

module.exports = {
  meta: {
    type:     'problem',
    docs:     {
      description: 'Require requireSession() as first statement in server actions',
      url:         'docs/runbooks/server-action-pattern.md',
    },
    schema:   [],
    messages: {
      missingRequireSession:
        'Server action "{{name}}" must call `requireSession()` as its first statement (CLAUDE.md §3, REQ-031). See docs/runbooks/server-action-pattern.md.',
    },
  },
  create(context) {
    let isUseServerFile = false

    function checkFunction(node) {
      if (!isUseServerFile) return
      if (!node.async) return

      const name = node.id?.name ?? node.parent?.id?.name ?? '(anonymous)'
      if (typeof name === 'string' && name.startsWith('_')) return  // exempt private helpers

      const body = node.body?.body ?? []
      const firstStatement = body[0]

      const isRequireSession =
        firstStatement?.type === 'VariableDeclaration' &&
        firstStatement.declarations?.[0]?.init?.type === 'AwaitExpression' &&
        firstStatement.declarations?.[0]?.init?.argument?.type === 'CallExpression' &&
        (
          firstStatement.declarations[0].init.argument.callee?.name === 'requireSession' ||
          (
            firstStatement.declarations[0].init.argument.callee?.type === 'MemberExpression' &&
            firstStatement.declarations[0].init.argument.callee.property?.name === 'requireSession'
          )
        )

      if (!isRequireSession) {
        context.report({
          node,
          messageId: 'missingRequireSession',
          data:      { name },
        })
      }
    }

    return {
      // Detect 'use server' directive at file top
      Program(node) {
        const directive = node.body?.[0]
        if (
          directive?.type === 'ExpressionStatement' &&
          directive.expression?.type === 'Literal' &&
          directive.expression.value === 'use server'
        ) {
          isUseServerFile = true
        }
      },
      // Cover both function declarations and arrow function expressions when exported
      'ExportNamedDeclaration > FunctionDeclaration': checkFunction,
      'ExportNamedDeclaration > VariableDeclaration > VariableDeclarator > ArrowFunctionExpression'(node) {
        // Wrap into a faux node that has the same shape checkFunction expects
        const fakeNode = {
          async:  node.async,
          body:   node.body,
          id:     node.parent?.id,
          parent: node.parent,
        }
        checkFunction(fakeNode)
      },
    }
  },
}
```
