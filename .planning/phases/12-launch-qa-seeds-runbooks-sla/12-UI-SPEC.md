# Phase 12 — UI Design Contract

> Visual and interaction contract for Phase 12: Launch + QA + Seeds + Runbooks + SLA.
> Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> Source: CONTEXT.md (D-06, D-07), REQUIREMENTS.md (REQ-156, REQ-415, REQ-416), phase_context,
> and codebase scan of packages/ui/tokens/* and packages/ui/src/dashboard/dashboard.css.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none — manual var(--mj-*) CSS token system (Phase 4) |
| Preset | not applicable |
| Component library | none — project-defined token-driven components |
| Icon library | none — inline SVG or CSS-only indicator shapes |
| Font | Agency font via `--mj-font-brand` (next/font/google per-app injection); fallback chain: `var(--font-brand, var(--mj-font-sans))` |

**Token discipline (mandatory for all Phase 12 surfaces):**
- Zero hex color literals. All color references use `var(--mj-*)` tokens. Enforced by Phase 4 AJV validator.
- Zero inline `style` attributes in admin surfaces — use external CSS classes referencing `var(--mj-*)` tokens (Pitfall 4.5: per-request CSP nonce blocks inline style attributes).
- Exception: public-facing SLA page (web-main) follows the existing privacy/page.tsx pattern which uses `style={{ ... }}` with `var(--mj-*)` values. This is acceptable for React Server Component pages outside the nonce scope.

## Spacing Scale

All values are multiples of 4px. References are to existing `--mj-space-*` tokens from `packages/ui/tokens/layer-4-layout.css`.

| Token | Value | Usage |
|-------|-------|-------|
| xs | `var(--mj-space-1)` = 4px | Icon gaps, badge padding, step connector dots |
| sm | `var(--mj-space-2)` = 8px | Compact element spacing, tag gaps |
| md | `var(--mj-space-4)` = 16px | Default element spacing, label-to-input gap |
| lg | `var(--mj-space-6)` = 24px | Section padding, card internal padding |
| xl | `var(--mj-space-8)` = 32px | Between wizard steps, between SLA sections |
| 2xl | `var(--mj-space-12)` = 48px | Major section breaks in SLA page |
| 3xl | `var(--mj-space-16)` = 64px | Page-level top/bottom padding |

Exceptions:
- Touch-target minimum for all buttons and interactive controls: 44px height (matches `ctaPrimary` pattern in privacy/page.tsx with `minHeight: '44px'`). Achieved via `min-height: 44px` plus `padding-y: var(--mj-space-3)`.
- Wizard step indicator dots: 8px diameter with 4px gap (`--mj-space-2` and `--mj-space-1`).

## Typography

Pre-populated from `packages/ui/src/dashboard/dashboard.css` which documents the locked contract: "exactly 4 sizes (14/16/24/36) and 2 weights (400/700)". Confirmed by STATE.md accumulated decisions.

| Role | Token | Size | Weight Token | Weight | Line Height Token | Line Height |
|------|-------|------|--------------|--------|-------------------|-------------|
| Label / caption | `--mj-text-size-sm` | 14px | `--mj-weight-normal` | 400 | `--mj-leading-normal` | 1.5 |
| Body | `--mj-text-size-base` | 16px | `--mj-weight-normal` | 400 | `--mj-leading-normal` | 1.5 |

Deviation rule: no additional sizes or weights may be introduced in Phase 12. If a visual need exists, use a size from this table with `var(--mj-color-text-secondary)` to de-emphasize.

## Color

Pre-populated from `packages/ui/tokens/layer-2-semantic-color.css` (Phase 4). All references are token names, never hex or OKLCH literals.

| Role | Token | Underlying Value | Usage |
|------|-------|-----------------|-------|
| Dominant (60%) | `var(--mj-color-bg-primary)` | `oklch(1 0 0)` — white | Page background, wizard card background, SLA page background |
| Secondary (30%) | `var(--mj-color-bg-secondary)` | `oklch(0.98 0 0)` — off-white | Wizard step panel bg, SLA table of contents box, QA report row stripe |
| Accent (10%) | `var(--mj-color-brand-500)` | `oklch(0.55 0.19 250)` — blue | See "Accent reserved for" list below |
| Destructive | `var(--mj-color-error)` | `oklch(0.63 0.21 25)` — red | Fail/error badges in QA report, ΔE warning in wizard, form validation errors |
| Success | `var(--mj-color-success)` | `oklch(0.72 0.17 142)` — green | Pass badges in QA report and Brand Setup Wizard steps |
| Warning | `var(--mj-color-warning)` | `oklch(0.83 0.17 78)` — amber | ΔE < 30 guardrail warning, skipped/pending QA check badge |
| Text primary | `var(--mj-color-text-primary)` | `oklch(0.12 0 0)` — near-black | All body text, heading text |
| Text secondary | `var(--mj-color-text-secondary)` | `oklch(0.55 0 0)` — mid-gray | Labels, captions, SLA sub-text, wizard step hints |
| Border default | `var(--mj-color-border-default)` | `oklch(0.90 0 0)` — light gray | Card borders, wizard step borders, form input borders |
| Focus ring | `var(--mj-color-border-focus)` | `oklch(0.55 0.19 250)` — brand blue | All focusable interactive elements |

**Accent reserved for (explicit list — never "all interactive elements"):**
1. Primary CTA button background in Brand Setup Wizard ("Save Brand Setup")
2. Active wizard step indicator dot and connector line
3. "Complete Setup" summary action button
4. Link color in SLA page body (`var(--mj-color-text-link)` = brand-500)
5. Focus ring on all inputs and buttons (`var(--mj-color-border-focus)` = brand-500)
6. Active tab underline in QA Report View (if tab navigation is present)

**Color NOT used for:**
- Warning badges (use `--mj-color-warning`)
- Error/fail states (use `--mj-color-error`)
- Informational text (use `--mj-color-text-secondary`)

## Surface 1: Brand Setup Wizard

**Route:** `/admin/brand-setup`
**Auth:** `requireSession()` as first call, then `if (session.role !== 'super_admin') redirect('/admin')`
**Pattern:** Payload custom admin view — same registration pattern as `DashboardView` (Plan 11-04)

### Layout

Single-column centered layout within Payload admin shell:
- Max-width: `var(--mj-container-sm)` = 640px, horizontally centered
- Page padding: `var(--mj-space-8)` top/bottom, `var(--mj-space-6)` horizontal

### Step Navigation

5 steps rendered as a horizontal progress indicator at the top of the wizard card:

| Step | Label |
|------|-------|
| 1 | Logo |
| 2 | Colors |
| 3 | Identity |
| 4 | API Keys |
| 5 | DNS + Warmup |

Step indicator anatomy:
- Dot: 8px circle. Completed = `var(--mj-color-success)` fill. Active = `var(--mj-color-brand-500)` fill. Pending = `var(--mj-color-border-default)` fill.
- Connector line: 1px solid. Completed segment = `var(--mj-color-success)`. Pending segment = `var(--mj-color-border-default)`.
- Step label: 14px / weight 400 / `var(--mj-color-text-secondary)`. Active step: `var(--mj-color-text-primary)` / weight 700.
- Step indicator sits 24px above the wizard card border.

### Wizard Card

- Background: `var(--mj-color-bg-primary)` with `1px solid var(--mj-color-border-default)` border
- Border radius: `var(--mj-radius-xl)` = 12px
- Internal padding: `var(--mj-space-8)` = 32px all sides
- Card shadow: `var(--mj-shadow-sm)` (from layer-5-effects)

### Step 1: Logo Upload

Fields:
- File input: accepts SVG, PNG, JPG. Label: "Upload your logo". Hint (14px secondary): "SVG files are sanitized automatically via DOMPurify + SVGO."
- Logo preview: 120px × 120px container, `var(--mj-radius-lg)` border, `var(--mj-color-bg-secondary)` background. Shows uploaded logo or a placeholder icon (inline SVG silhouette) in `var(--mj-icon-secondary)`.
- Security note (14px / secondary): "Your SVG will be sanitized before saving. Malicious code is removed automatically."

States:
- Idle: dashed border `var(--mj-color-border-default)`, drag-and-drop hint text
- Uploading: animated progress bar, background `var(--mj-color-bg-secondary)`, `var(--mj-color-brand-500)` fill progress
- Success: `var(--mj-color-success)` border, checkmark indicator
- Error: `var(--mj-color-error)` border, error message 14px / `var(--mj-color-error)`

### Step 2: Brand Color Picker

Fields:
- Color input (`<input type="color">`) labeled "Primary brand color". Current value shown as a 24×24px swatch with hex value display (read-only text field).
- ΔE guardrail result: shown as a badge after color selection.

ΔE badge states:
- `ΔE ≥ 30` (sufficient contrast against seeded imagery): `var(--mj-color-success)` border + text, label "Color contrast OK"
- `ΔE < 30` (niche confusion risk): `var(--mj-color-warning)` border + text, label "Color too similar to niche imagery — consider a higher-contrast brand color"

Badge anatomy: `var(--mj-space-1)` `var(--mj-space-2)` padding, `var(--mj-radius-full)` border radius, 14px text, weight 400.

Hint (14px / secondary): "We check your color against your niche's pre-seeded imagery to ensure visual distinction. ΔE < 30 means your brand may look too similar to generic imagery."

### Step 3: Identity Fields

Fields (all required):
- Tagline — text input, maxlength 80, placeholder "e.g. Grow Faster with Smarter Ads"
- About — textarea, rows 4, maxlength 500, placeholder "Describe your agency in 1-2 sentences"
- Phone — tel input, placeholder "+1 (555) 000-0000"
- Address — textarea, rows 2, placeholder "Street, City, State ZIP"

Field layout: single column, label above input, `var(--mj-space-4)` gap between fields.
Required indicator: asterisk `*` in `var(--mj-color-error)`, 14px, after label text.

### Step 4: API Keys

Fields (all optional):
- GA4 Measurement ID — text input, placeholder "G-XXXXXXXXXX"
- Microsoft Clarity Project ID — text input, placeholder "xxxxxxxxxx"
- Meta Pixel ID — text input, placeholder "000000000000000"

Security banner (shown above fields):
- Background: `var(--mj-color-bg-secondary)`
- Border: `1px solid var(--mj-color-border-default)`
- Border radius: `var(--mj-radius-md)`
- Padding: `var(--mj-space-4)`
- Text (14px / secondary): "API keys are stored in Doppler and are never exposed in page source or NEXT_PUBLIC_ variables."

Field inputs: `type="password"` with show/hide toggle button (inline SVG eye icon, 16×16px, `var(--mj-icon-secondary)`).

### Step 5: DNS + Warmup Checklist

Checklist items rendered as an ordered list:

| # | Item | Type |
|---|------|------|
| 1 | Add your custom domain's CNAME/A record pointing to your deployment | Manual |
| 2 | Confirm SSL certificate issued (auto-renews via Cloudflare) | Auto-verify |
| 3 | Email DKIM/SPF/DMARC records added (use /admin/email-setup to verify) | Link |
| 4 | GA4 stream is receiving data | Manual |
| 5 | Test form submission creates a CRM contact | Manual |
| 6 | Email warmup sequence is in DRAFT mode (activates after 35 days) | Status |

Item anatomy:
- Checkbox: 20px × 20px. Checked = `var(--mj-color-success)` background. Unchecked = `var(--mj-color-border-default)` border.
- Label: 16px / weight 400. Completed items: `var(--mj-color-text-secondary)` with strikethrough.
- Sub-hint: 14px / `var(--mj-color-text-secondary)`.

Link items open in the same tab (admin context). Auto-verify items show a "Verify now" button (`var(--mj-color-brand-500)`, secondary button style).

### Navigation Controls (all steps)

Footer bar within the wizard card:
- "Back" button: text-only, 16px / weight 400, `var(--mj-color-text-secondary)`, no border. Hidden on step 1.
- "Next" / "Save Brand Setup" button: primary button style — `var(--mj-color-brand-500)` background, `var(--mj-color-text-inverse)` text, `var(--mj-radius-md)` radius, `var(--mj-btn-padding-x)` `var(--mj-btn-padding-y)` padding, 14px / weight 600 (uses `--mj-btn-font-weight` = semibold). Min-height 44px.
- Step 5 replaces "Next" with "Save Brand Setup" (primary button, same style).
- "Skip for now" text link: 14px / secondary, only shown on optional steps (4 and 5). Navigates to next step.

### Credits Policy Section

Definition list: credit tiers keyed to downtime duration. Same `<dl>` style as Recovery Objectives.

## Interaction States

All interactive elements must declare all four states:

| State | Visual |
|-------|--------|
| Default | Per component spec above |
| Hover | `opacity: 0.85` on primary buttons. Border color shifts to `var(--mj-color-brand-500)` on inputs. |
| Focus | `outline: 2px solid var(--mj-color-border-focus)`, `outline-offset: 2px`. Never remove focus ring. |
| Disabled | `opacity: 0.4`, `cursor: not-allowed`. Wizard "Next" button disabled when required fields empty. |
| Loading | Primary button shows inline spinner (12px animated SVG circle, `var(--mj-color-text-inverse)`) + text "Saving…" |
| Error | Input border `var(--mj-color-error)`, error message below input at 14px / `var(--mj-color-error)` |
| Success | Input border `var(--mj-color-success)`, checkmark inline (12px SVG, `var(--mj-color-success)`) |

## Accessibility Contract

- All form inputs have `<label>` with `for` attribute matching input `id`.
- Required fields have `aria-required="true"` and visible `*` indicator.
- Error messages have `role="alert"` so screen readers announce validation failures.
- Wizard step indicator has `aria-label="Step N of 5: [Step Name]"`.
- Color is never the sole indicator of state (badges include text label, not just color).
- WCAG 2.2 AA minimum contrast on all text/background combinations using `var(--mj-*)` tokens.
- Minimum touch target 44px height on all buttons and checkboxes.
- QA matrix table has `<th scope="col">` on all column headers and `<th scope="row">` on agency name.
- SLA severity matrix table has same `scope` attributes.
- Focus order follows DOM order in wizard (step indicator → form fields → navigation controls).
