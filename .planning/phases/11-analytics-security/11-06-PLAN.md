# Plan 11-06: Cloudflare WAF + Security Rules — Managed Ruleset + Bot Fight + Per-Route Rate Limits via Terraform

## Objective

Configure Cloudflare WAF and rate limit rules across all 13 zones (`mjagency.com` + 12 agency subdomains) via Terraform — repeatable, version-controlled, reviewable.

Deliverables:
- **Managed Ruleset** — Cloudflare's OWASP Core Rule Set (free tier).
- **Bot Fight Mode** — challenges automated traffic from known-bad ASNs (free tier).
- **Custom firewall rules** — block known content scrapers, US-only geo challenge (CCPA scope per PROJECT.md), block requests with `x-middleware-subrequest` header (CVE-2025-29927 layer 1 defense).
- **Per-route rate limits** — 5 endpoint categories per RESEARCH §8 + UI-SPEC Surface 3 erasure form anti-abuse spec.
- **CI gate** — GitHub Action runs `terraform plan -detailed-exitcode` on PRs touching infra; manual `terraform apply` on merge.
- **Runbook** — phased rollout procedure (log-only → progressive enforcement) with false-positive triage workflow.

Purpose: REQ-146 (Cloudflare WAF + security rules). Wave-2 plan dependent on Plan 11-07 (rate limits include /api/csp-report and /api/rum endpoints introduced by 11-07; rate limit on /api/ccpa/* depends on Plan 11-05 introducing those routes — but planning these in parallel is safe because Terraform applies after deploy).

Output: `infra/cloudflare/` Terraform directory (new), GitHub Actions plan workflow, rollout runbook.

**Step 3 — Create infra/cloudflare/firewall-rules.tf.**

```hcl
# infra/cloudflare/firewall-rules.tf
# Custom expressions: known scraper block + US-only geo challenge + CVE-2025-29927 layer 1.

resource "cloudflare_ruleset" "custom_firewall" {
  for_each    = var.zone_ids
  zone_id     = each.value
  name        = "Custom Firewall — Plan 11-06"
  description = "REQ-145 + CVE-2025-29927 layer 1 + US-only geo + scraper block"
  kind        = "zone"
  phase       = "http_request_firewall_custom"

  # Rule 1 — Block known content scrapers
  rules {
    action      = var.enable_enforcing ? "block" : "log"
    expression  = "(http.user_agent contains \"AhrefsBot\") or (http.user_agent contains \"SemrushBot\") or (http.user_agent contains \"MJ12bot\")"
    description = "Block known content scrapers (AhrefsBot, SemrushBot, MJ12bot)"
  }

  # Rule 2 — CVE-2025-29927 layer 1 — strip x-middleware-subrequest header attacks
  rules {
    action      = var.enable_enforcing ? "block" : "log"
    expression  = "any(http.request.headers[\"x-middleware-subrequest\"][*] != \"\")"
    description = "Block requests with x-middleware-subrequest header (CVE-2025-29927 layer 1 — see docs/runbooks/cloudflare-edge.md)"
  }

  # Rule 3 — US-only geo (CCPA scope per PROJECT.md). Pitfall 8.1: Challenge, not Block.
  rules {
    action      = var.enable_enforcing ? "managed_challenge" : "log"
    expression  = "(ip.geoip.country ne \"US\")"
    description = "Challenge non-US visitors (Pitfall 8.1 — Challenge allows VPN users)"
  }

  # Rule 4 — Skip Bot Fight Mode for verified bots (Pitfall 8.4)
  rules {
    action      = "skip"
    expression  = "(cf.client.bot)"
    description = "Skip security checks for Cloudflare-verified bots (GoogleBot, Bingbot, etc.)"
    action_parameters {
      ruleset = "current"
    }
  }
}
```

```hcl
# infra/cloudflare/rate-limits.tf
# Per-route rate limits per RESEARCH §8.

resource "cloudflare_ruleset" "rate_limits" {
  for_each    = var.zone_ids
  zone_id     = each.value
  name        = "Rate Limits — Plan 11-06"
  description = "REQ-145 — per-route rate limits (auth, public forms, RUM, CSP report, public reads)"
  kind        = "zone"
  phase       = "http_ratelimit"

  # Auth attempts — Challenge at 10/min/IP
  rules {
    action      = var.enable_enforcing ? "managed_challenge" : "log"
    expression  = "(http.request.uri.path eq \"/api/auth/login\") or (http.request.uri.path eq \"/api/auth/refresh\") or (starts_with(http.request.uri.path, \"/api/sso/\"))"
    description = "Auth: 10/min/IP Challenge"
    ratelimit {
      characteristics     = ["ip.src"]
      period              = 60
      requests_per_period = 10
      mitigation_timeout  = 600
    }
  }

  # Public form posts — Block at 5/min/IP
  rules {
    action      = var.enable_enforcing ? "block" : "log"
    expression  = "(http.request.method eq \"POST\") and ((http.request.uri.path eq \"/api/contact\") or (http.request.uri.path eq \"/api/privacy/erasure-request\") or (http.request.uri.path eq \"/api/privacy/verify\") or (http.request.uri.path eq \"/api/ccpa/opt-out\"))"
    description = "Public form POSTs: 5/min/IP Block (anti-abuse)"
    ratelimit {
      characteristics     = ["ip.src"]

      requests_per_period = 5
      mitigation_timeout  = 3600

  }

  rules {
```

**Step 1 — Create .github/workflows/cloudflare-terraform-plan.yml.**

```yaml
name: Cloudflare Terraform Plan
on:
  pull_request:
    paths:
      - 'infra/cloudflare/**'

jobs:
  terraform-plan:
    runs-on: ubuntu-latest
    timeout-minutes: 10
    permissions:
      contents: read
      pull-requests: write
    steps:
      - uses: actions/checkout@v4
      - uses: hashicorp/setup-terraform@v3
        with:
          terraform_version: '1.6.6'
      - name: Terraform fmt check
        working-directory: infra/cloudflare
        run: terraform fmt -check -recursive
      - name: Terraform init (no backend)
        working-directory: infra/cloudflare
        run: terraform init -backend=false
      - name: Terraform validate
        working-directory: infra/cloudflare
        run: terraform validate
      - name: Terraform plan (detailed exit code)
        working-directory: infra/cloudflare
        env:
          CLOUDFLARE_API_TOKEN: ${{ secrets.CLOUDFLARE_API_TOKEN }}
          TF_VAR_account_id: ${{ secrets.CLOUDFLARE_ACCOUNT_ID }}
          TF_VAR_zone_ids: ${{ secrets.CLOUDFLARE_ZONE_IDS_JSON }}
```
