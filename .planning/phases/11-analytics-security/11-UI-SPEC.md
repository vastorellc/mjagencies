| Token | Value | Justification |
|-------|-------|--------------|

## Color

All values are `var(--mj-*)` references. Zero hex literals. OKLCH semantic tokens from
`packages/ui/tokens/layer-2-semantic-color.css` and `layer-6-components.css`.

### Color Contract (60/30/10)

| Role | CSS Token | Phase 11 Usage |
|------|-----------|---------------|
| Dominant (60%) — surface | `var(--mj-color-bg-primary)` | Dashboard page background, privacy page body, erasure form container, modal background, cookie hint banner background |
| Secondary (30%) — structure | `var(--mj-color-bg-secondary)` | Dashboard KPI card background, dashboard sidebar/tab strip, modal overlay backdrop tint, privacy page section alternation, footer area background where opt-out link sits |
| Accent (10%) — brand | `var(--mj-color-brand-500)` | See reserved-for list below |
| Destructive | `var(--mj-color-error)` | Opt-out modal "Stop Tracking and Clear My Data" CTA, erasure form irreversibility warning copy, dashboard error states (failed GA4 fetch) |
| Success | `var(--mj-color-success)` | Erasure email-sent confirmation, dashboard "all systems healthy" indicator, opt-out completion confirmation |
| Warning | `var(--mj-color-warning)` | Erasure verification expiry countdown (link expires in N hours), RUM threshold breach badge (LCP > 2.5s, INP > 200ms, CLS > 0.1), CSP report-only mode banner |
| Info | `var(--mj-color-info)` | Cookie hint banner accent strip, privacy page "data systems we use" callout, dashboard "near-realtime — last refresh 12s ago" indicator |

**Accent (`var(--mj-color-brand-500)`) reserved exclusively for:**

1. Erasure form primary CTA "Send Verification Email" (button background)
2. Privacy page primary CTA "Request Data Deletion" (button background)
3. Opt-out modal secondary CTA "Keep Current Settings" border + text
4. Dashboard active tab indicator underline (per-agency vs platform-overview)
5. Focus ring on all interactive elements (`outline-color: var(--mj-color-border-focus)` which resolves to `var(--mj-color-brand-500)`)
6. Inline text links inside privacy/erasure pages (`color: var(--mj-color-text-link)`)

**Accent must NOT appear on:** destructive CTAs (use `--mj-color-error`), KPI metric values
(use `--mj-color-text-primary`), success states (use `--mj-color-success`), card borders,
cookie hint banner background, footer opt-out link in normal state (uses `--mj-color-text-secondary`).

> Per-agency themes override `--mj-color-brand-500` at `[data-agency="..."]` layer (Plan 04-04).
> All 12 OKLCH niche themes apply automatically. brand.com uses default brand palette. The
> Payload admin context inherits brand.com tokens (Phase 5).

### Visuals — Surface 1

- KPI card: white background (`var(--mj-color-bg-primary)`) on dashboard background
  (`var(--mj-color-bg-secondary)`). Card padding `var(--mj-space-6)`. Border-radius
  `var(--mj-radius-lg)`. Border: 1px solid `var(--mj-color-border-default)`.
- KPI metric value: Display role (36px / 700 / `var(--mj-color-text-primary)`).
- KPI label: Label role (14px / 400 / `var(--mj-color-text-secondary)`).
- Trend delta positive: `var(--mj-color-success)`. Negative: `var(--mj-color-error)`.
- Sparkline: 24px tall, full card width, `var(--mj-color-brand-500)` stroke at
  `stroke-width: 1.5px`. No fill.
- Tabs: underline indicator (2px, `var(--mj-color-brand-500)`) on active tab. Inactive tab
  text uses `var(--mj-color-text-secondary)`.
- Top pages table: striped rows using `var(--mj-color-bg-secondary)` zebra. Row padding
  `var(--mj-space-2)` `var(--mj-space-4)`.
- RUM threshold pills: green (`var(--mj-color-success)`) when LCP ≤ 2.5s, INP ≤ 200ms,
  CLS ≤ 0.1; warning (`var(--mj-color-warning)`) when above. Pill padding `var(--mj-space-1)`
  `var(--mj-space-2)`, border-radius `var(--mj-radius-full)`.

### Interactions — Surface 1

- Polling: client component `useDashboardPolling()` hook fires fetch every 60,000 ms only
  while `document.visibilityState === 'visible'`. Pauses on tab blur. Resumes on focus.
- Refresh button: clicking triggers immediate fetch (no debounce); icon spins for duration
  (`<Loader2 className="animate-spin" />` from lucide). Shows toast "Refresh complete" on
  success — toast pattern from Phase 10 Puck builder.
- Tab switch (super_admin only): `<a href="/admin/dashboard?view=platform">` — full
  navigation, server re-renders. No client-side route swap (avoids stale per-agency state).
- KPI card click: navigates to detailed view (`/admin/dashboard/traffic`,
  `/admin/dashboard/leads`, etc. — out of scope for Phase 11 v1, cards are read-only display).

### Accessibility — Surface 1

- Page has single `<h1>Dashboard</h1>`. KPI card labels are `<h2>` semantically; styled as
  Label role visually but rendered as `<h2 class="sr-only">` plus a separate display-styled
  `<span aria-hidden="true">` if needed for the design — OR use `<h2>` styled as Label
  (preferred, simpler).
- Tabs: `role="tablist"`, each tab `role="tab"` with `aria-selected="true|false"` and
  `aria-controls` pointing to the panel ID.
- Refresh status uses `aria-live="polite"` so screen readers announce "Last refresh: 12s ago"
  changes without interruption.
- Sparkline `<svg>` includes `<title>` element: e.g. "Sessions trend, last 7 days, peaked
  Wednesday at 2,140".
- Color contrast on all text meets WCAG 2.2 AA (4.5:1 body, 3:1 large) — already enforced by
  Phase 4 OKLCH tokens.
- Focus ring: 2px solid `var(--mj-color-border-focus)` with 2px offset on all interactive
  elements (refresh button, tabs, KPI cards if clickable).

### Error / Empty / Loading States — Surface 1

| State | Trigger | Visual |
|-------|---------|--------|
| Loading (initial) | First render before SSR data hydrates | Skeleton boxes for each KPI card; no copy. Background `var(--mj-color-bg-secondary)` with shimmer animation. |
| Loading (refresh) | Polling tick or manual refresh in flight | Refresh icon spins; existing values remain visible; aria-live announces "Refreshing dashboard data..." |
| Empty (new agency) | All metrics return zero / null | Single centered card with empty-state heading + body copy. Hides all KPI cards. |
| Partial error (GA4 down) | GA4 Data API call fails; Postgres + web-vitals succeed | Traffic card shows error inline message ("Traffic data temporarily unavailable"); other KPI cards render normally. |
| Total error | All data sources fail | Centered card: error heading + body. Refresh button visible. |

## Cross-Surface Design System Constraints

### Type Constraints (enforced)

- Maximum 4 type sizes across all 6 surfaces: 14px / 16px / 24px / 36px.
- Maximum 2 weights across all 6 surfaces: 400 / 700.
- No `medium` (500) or `semibold` (600) anywhere in Phase 11. (Phase 9 used 4 weights for
  legacy block compatibility — Phase 11 components are net-new and apply the strict 2-weight
  rule.)

### Color Constraints (enforced)

- All colors expressed as `var(--mj-color-*)` references. Zero hex literals. Zero raw OKLCH.
- 60/30/10 split:
  - 60% `var(--mj-color-bg-primary)` — backgrounds
  - 30% `var(--mj-color-bg-secondary)` — cards, dashboard structure, table zebra
  - 10% `var(--mj-color-brand-500)` — accents (per-surface reserved-for list above)
- Destructive uses `var(--mj-color-error)` only (CCPA confirm, dashboard fetch errors).
- Semantic state colors (success / warning / info) use their respective semantic tokens.

## Copywriting Tone Guide

### Three voice contexts

| Context | Tone | Example |
|---------|------|---------|
| Payload admin (Surfaces 1–2) | Operator-direct: short, factual, status-oriented | "Last refresh: 12s ago", "Healthy", "Above target" |
| Public privacy/erasure (Surfaces 3, 5) | Plain-language legal: clear, no jargon, sentences ≤20 words | "California residents have the right to request deletion of their personal information under the CCPA." |
| Public hint/modal (Surfaces 4, 6) | Respectful and brief: present tense, action-oriented, no marketing language | "We use analytics to improve this site." |

### CCPA exact-wording requirements (do not paraphrase)

- Footer link: "Do Not Sell or Share My Personal Information"
  - Source: California Civil Code §1798.135(a)(1) — the AG Final Regulations define this
    exact link text as the conforming opt-out signal.
- 30-day SLA: "within 30 days" — explicit, do not soften to "as soon as possible" or "within
  a month".
- Erasure scope: must explicitly enumerate (1) categories of data deleted and (2) categories
  excluded under legal hold. (Surface 3 + Surface 5 both comply.)

### CTA copywriting rules

| Rule | Example | Counter-example |
|------|---------|-----------------|
| Use specific outcome verbs | "Send Verification Email" | "Submit" ❌ |
| Name what's being kept on cancel | "Keep Current Settings" | "Cancel" ❌ |
| Name what's being acted on (destructive) | "Stop Tracking and Clear My Data" | "Confirm" ❌ |
| Brand-name back links | "Return to MJ Agency" | "Go Back" ❌ |
| Acceptable casual on dismiss | "Got It" | (only on non-blocking dismissable hint) |
| Privacy-page CTA reflects action | "Request Data Deletion" | "Click Here" ❌ |
| Opt-out modal opener (non-destructive opener) | "Open Tracking Settings" | "Settings" ❌ |
| Re-send action | "Resend Verification Email" | "Resend" alone ❌ |
| Re-enable tracking | "Re-enable Tracking" | "Turn On" ❌ |

### Empty state pattern (consistent across surfaces)

Three lines:
1. Heading: state of affairs (positive framing)
2. Body: what causes data to populate
3. Optional next-step CTA

Example (Surface 1):
- "Your dashboard will populate within 24 hours" (heading)
- "Once visitors begin reaching your site, traffic, RUM, and lead metrics will appear here.
  CRM and revenue numbers update in real time as deals progress." (body)
- (no CTA — wait state)

### Error state pattern (consistent across surfaces)

Two lines:
1. What went wrong (specific, no internal jargon)
2. What to do next (concrete action)

Example (Surface 3): "We could not process your request right now. Please try again, or email
privacy@[agency-domain].com."

## Security Notes (enforced at component layer)

- **CSP nonce (D-09):** Per-request nonce generated in `middleware.ts` and read via
  `headers()` in layout. All inline `<style>` and `<script>` in Phase 11 surfaces consume
  the nonce via `<NonceProvider>` from Phase 11-07.
- **No `dangerouslySetInnerHTML`** on any Phase 11 component. Privacy page body copy is
  rendered via React-native JSX, not from CMS-stored HTML strings.
- **`mj_consent` cookie attributes (D-02):** `Path=/`, `Max-Age=31536000` (1 year), `SameSite=Lax`,
  `Secure`, NOT httpOnly (client must read for `useConsent()` hook). Cookie content is non-PII
  enum (`tracking_allowed` | `tracking_blocked`).
- **`mj_consent_hint_dismissed` cookie:** Same attributes as `mj_consent`. Non-PII flag.
- **Erasure form (Surface 3):** No `requireSession()` (public form). Server action validates
  email format, rate-limits 5 req/IP/hour (Cloudflare WAF), generates email-verification
  token via Phase 3 helper, enqueues BullMQ job with `sensitiveData: true` (encrypted).
- **Opt-out modal (Surface 4):** Server endpoint `/api/ccpa/opt-out` accepts no parameters
  beyond cookie state. Idempotent — multiple calls don't re-fire deletion APIs (deduped via
  Redis key).
- **Dashboard data fetch (Surfaces 1–2):** Server action `loadDashboardMetrics(agencyId)`
  begins with `requireSession()` per CLAUDE.md rule 3. Asserts `session.agencyId === agencyId`
  for per-agency view; asserts `session.role === 'super_admin'` for platform view. RLS
  enforces tenant isolation at DB level as second layer of defense.
- **No secrets in `NEXT_PUBLIC_*`:** GA4 Measurement ID is OK to expose; GA4 Service Account
  JSON for Data API stays server-side only. Microsoft Clarity project ID is OK to expose;
  Clarity Delete API token stays server-side only.
