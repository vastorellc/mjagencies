# Plan 11-07: Security Hardening — CSP Nonce Per-Request + Report Endpoint + OWASP ZAP Scan

## Objective

Foundational Phase 11 wave-1 work. Three deliverables, all entangled:

1. **CSP nonce middleware** — replace the static CSP in `packages/auth/src/security-headers.ts` with a per-request nonce-CSP generated in `packages/auth/src/middleware.ts` (Edge runtime, jose-only path). Stage 1 = `Content-Security-Policy-Report-Only`; flip to enforcing later via `CSP_ENFORCING=true`.
2. **Persistence tables** — create `csp_reports` (platform-wide) and `web_vitals` (per-agency, RLS) Postgres tables NOW so Wave 2 plans (especially 11-04 dashboard) can read from them. Includes `/api/csp-report` and `/api/rum` POST endpoints in all 13 apps. **Updates `WebVitalsReporter` (Phase 8) to dual-emit**: existing `window.gtag()` preserved + new `navigator.sendBeacon('/api/rum', ...)` added per RESEARCH.md Critical Discovery.
3. **OWASP ZAP CI gates** — per-PR baseline scan blocking on high-severity, weekly cron full sweep against staging, ZAP rules.tsv tuning, and a CI grep gate preventing CSP regression in `security-headers.ts`.

Purpose: REQ-145 (CSP nonce per-request) and REQ-147 (OWASP ZAP zero high-severity). This MUST run first because all Wave 2 plans (GA4, Clarity, opt-out, erasure, WAF) need the CSP allowlist established here. Plan 11-04 (dashboard) reads from `web_vitals` — that table only exists after this plan.

Output: Edge-safe nonce CSP, two new Postgres tables with migration, 26 new API route files (13 csp-report + 13 rum), updated WebVitalsReporter, NonceProvider context, and 3 GitHub Actions workflows.

**Step 2 — Extend packages/auth/src/middleware.ts with nonce generation.**

Inside the middleware function (after agency extraction, before/after `applySecurityHeaders` call), add nonce CSP injection. Use `crypto.randomUUID()` (Web Crypto API, Edge-safe per CLAUDE.md rule 4). Set the nonce on the request headers (so layout can read via `headers()`) AND on the response headers as the CSP value.

```typescript
// packages/auth/src/middleware.ts — INSIDE the createAuthMiddleware factory function
// After agency extraction; applies to BOTH the redirect path and the next() path.

const nonce = crypto.randomUUID()
const slugHeader = host.split('.')[0] // for report tagging — best-effort

const cspDirectives = [
  "default-src 'self'",
  `script-src 'self' 'nonce-${nonce}' 'strict-dynamic' https://www.googletagmanager.com https://www.clarity.ms`,
  `style-src 'self' 'nonce-${nonce}'`,
  "img-src 'self' data: https://imagedelivery.net https://www.google-analytics.com",
  "connect-src 'self' https://api.cloudflare.com https://www.google-analytics.com https://*.clarity.ms https://cloudflareinsights.com",
  "frame-src 'self' https://js.stripe.com",
  "frame-ancestors 'self'",
  "base-uri 'self'",
  "form-action 'self'",
  "object-src 'none'",
  "report-uri /api/csp-report",
].join('; ')

// Propagate nonce to layout via x-nonce request header
const requestHeaders = new Headers(req.headers)
requestHeaders.set('x-nonce', nonce)

// Build response that forwards updated request headers (Next.js 15 App Router pattern — RESEARCH Pitfall 7.6)
const response = NextResponse.next({ request: { headers: requestHeaders } })

// D-08 Stage 1 — Report-Only at launch; flip via env var post-launch
const cspHeaderName = process.env.CSP_ENFORCING === 'true'
  ? 'Content-Security-Policy'
  : 'Content-Security-Policy-Report-Only'
response.headers.set(cspHeaderName, cspDirectives)

applySecurityHeaders(response) // remaining 6 headers (no CSP)
return response
```

Apply this to BOTH outgoing branches: the public-path next(), the authenticated next(), AND the redirect-to-/login response (the redirect path also needs CSP because the /login page has scripts). Place nonce generation as the FIRST step inside the middleware function so all branches use the same nonce per request.

 * REQ-146 D-08: receives Content-Security-Policy-Report-Only violation reports.

 * Pitfall 9.1: skips ZAP scanner User-Agent to avoid table pollution.

import { db, cspReports } from '@mjagency/db'

export const runtime = 'nodejs' // postgres-js requires Node, not Edge

const AGENCY_SLUG = '{slug}' // e.g., 'web-ecommerce' — replace per app

export async function POST(req: Request): Promise<Response> {
  // Pitfall 9.1: drop ZAP scanner reports
  const ua = req.headers.get('user-agent') ?? ''
  if (/ZAP\//i.test(ua)) {
    return new Response(null, { status: 204 })

  //   { "csp-report": { "document-uri": ..., "blocked-uri": ..., ... } }

    const parsed = JSON.parse(text)
    report = parsed['csp-report'] ?? parsed

    return new Response(null, { status: 204 }) // ignore malformed

    blockedUri: report['blocked-uri'] ?? null,
    violatedDirective: report['violated-directive'] ?? null,
    originalPolicy: report['original-policy'] ?? null,
    sourceFile: report['source-file'] ?? null,
    lineNumber: typeof report['line-number'] === 'number' ? report['line-number'] : null,

  })

const AGENCY_SLUG = '{slug}' // e.g., 'web-ecommerce'
const AGENCY_ID = process.env.NEXT_PUBLIC_AGENCY_ID! // UUID, set per-app at build

interface RumPayload {
  metric_name: 'LCP' | 'INP' | 'CLS' | 'FCP' | 'TTFB'
  value: number
  rating?: string
  navigation_type?: string
  page_path: string

const ALLOWED_METRICS = new Set(['LCP', 'INP', 'CLS', 'FCP', 'TTFB'])

export async function POST(req: Request): Promise<Response> {
  let payload: RumPayload
  try {
    payload = await req.json() as RumPayload
  } catch {

  if (typeof payload.page_path !== 'string' || payload.page_path.length > 500) {

    pagePath: payload.page_path,
    metricName: payload.metric_name,
    value: payload.value,
    rating: payload.rating ?? null,
    navigationType: payload.navigation_type ?? null,
  })
