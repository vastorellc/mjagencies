**Token rule (mandatory, non-negotiable):** Every React component and page file in Phase 9
must use ONLY `var(--mj-*)` CSS custom properties. Zero hex literals. Zero hardcoded OKLCH
values. Zero inline `color:`, `background-color:`, `font-size:`, `margin:`, or `padding:`
with raw values. Violations are caught by the Phase 4 AJV validator CI gate.

## Page Specifications

### Contact Form Page (Modified — state layer added)

**Route:** `/contact` (exists in all 12 agency apps)
**Form fields (existing):** Full name, Email address, Message (all required)
**New in Phase 9:** Client-side validation on submit, inline field errors, success/error banners

#### Form Field Anatomy

Each form field group follows this DOM structure:
```
<div> (field wrapper)
  <label htmlFor="...">Label text <span aria-hidden="true" style="color: var(--mj-color-error)">*</span></label>
  <input ... aria-required="true" aria-invalid="true|false" aria-describedby="{id}-error" />
  <span id="{id}-error" role="alert" style="color: var(--mj-color-error); fontSize: var(--mj-text-size-sm)">
    {error message — see Copywriting Contract below}
  </span>
</div>
```

**Accessibility mandates:**
- `aria-invalid="true"` set on the `<input>` / `<textarea>` when field has an error
- `aria-describedby="{fieldId}-error"` pointing to the error span — present always, content populated on error
- Error span uses `role="alert"` so screen readers announce it on insertion
- Error span is hidden (empty string content) when field is valid — do NOT use `display: none` or `aria-hidden` on the span itself; instead render empty content (`""`) so the DOM node is always present and live region works correctly
- Focus moves to the first invalid field after failed submit attempt

#### Submit Button States

| State | Visual | DOM |
|-------|--------|-----|
| Idle | Background: `var(--mj-btn-bg-primary)` | `type="submit"` |
| Submitting | Background: `var(--mj-btn-bg-primary)` at `opacity: 0.7`; button text: "Sending..." | `disabled` attribute; `aria-busy="true"` |
| Post-success | Button hidden — success banner shown instead | `aria-live="polite"` region |
| Post-error | Button re-enabled — error banner shown | `aria-live="polite"` region |

#### Success Banner

Displayed after successful POST 200. Replaces the form entirely (form unmounts, banner mounts).

```
<div role="status" aria-live="polite" style="
  padding: var(--mj-space-6);
  backgroundColor: var(--mj-color-bg-secondary);
  borderRadius: var(--mj-radius-xl);
  borderLeft: 4px solid var(--mj-color-success);
">
  <p style="fontWeight: var(--mj-weight-semibold); color: var(--mj-color-text-primary)">
    Message sent
  </p>
  <p style="color: var(--mj-color-text-secondary); marginTop: var(--mj-space-2)">
    {success copy — see Copywriting Contract}
  </p>
</div>
```

#### API Error Banner

Displayed after a non-200 response from `/api/contact`. Form remains visible and re-enabled.

```
<div role="alert" aria-live="assertive" style="
  padding: var(--mj-space-4) var(--mj-space-6);
  backgroundColor: var(--mj-color-bg-primary);
  border: 1px solid var(--mj-color-error);
  borderRadius: var(--mj-radius-md);
  marginBottom: var(--mj-space-6);
">
  <p style="color: var(--mj-color-error); fontWeight: var(--mj-weight-medium)">
    {api error copy — see Copywriting Contract}
  </p>
</div>
```

#### Layout Structure

```
<page root> padding: var(--mj-space-16) var(--mj-space-6); maxWidth: var(--mj-container-md); margin: 0 auto
  <skip link>           (same pattern as contact page — WCAG 2.4.1)

      <div id="cal-embed-wrapper">
        <!-- Cal.com inline embed renders here -->
      </div>
    </section>
    <section> (trust copy below embed, optional per agency)
      Content from existing block library — HeroMinimal or ContentTwoCol
    </section>
  </main>
```

#### Cal.com Embed Contract

**Embed type:** Inline embed (not a modal, not a floating button). Cal.com renders its
calendar UI inline on the page using `@calcom/embed-react` or the Cal.com embed snippet.

**White-labeling:** Each agency configures its Cal.com theme via the Cal.com admin UI.
The embed receives `theme` and `brandColor` props set to the agency's `--mj-color-brand-500`
value. This is a runtime prop — computed server-side from `NICHE_PALETTES[agencySlug].primary500`.

**Embed wrapper dimensions:**
- Width: 100% of container (max `var(--mj-container-md)` = 768px)
- Min-height: 600px — prevents layout shift while Cal.com embed loads
- Background: `var(--mj-color-bg-primary)` — visible during embed load

**Loading state:** Set a `min-height: 600px` on the wrapper div before the embed script
loads. No custom spinner — the Cal.com embed renders its own skeleton loader.

**Cal.com embed does NOT use `var(--mj-*)` tokens internally.** The wrapper div uses tokens
for outer spacing only. The embed iframe/inline content is fully Cal.com's UI.

**Namespace convention (per-agency isolation):** Each agency uses a unique Cal.com namespace
matching the agency slug: `cal-{agencySlug}` (e.g., `cal-ecommerce`, `cal-growth`). This
enables per-agency booking page configuration.

**`<script>` placement:** The Cal.com embed initialization script is placed as the last
child of the booking page's `<body>` (via `next/script` with `strategy="lazyOnload"`) to
avoid blocking LCP. The embed wrapper must have `min-height: 600px` set before the script
loads to prevent CLS.

**IMPORTANT — no `dangerouslySetInnerHTML` for Cal.com embed.** Use `next/script` + the
Cal.com embed API. If Cal.com requires inline script initialization, it must go through
`next/script` with `id` attribute and `strategy="afterInteractive"`.
