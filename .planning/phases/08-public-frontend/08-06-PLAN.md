## Goal

Build a reusable `runAxeTest()` helper in `packages/testing` and wire it into axe-core test suites for the agency apps. Tests run in jsdom via Vitest, assert zero `impact === 'critical'` violations, and check required WCAG 2.2 AA structural elements (skip link, semantic landmarks, focus ring presence). One representative test suite is created for `apps/web-ecommerce`; the pattern is identical for all other agencies.

## Feature

**Name:** axe-core WCAG 2.2 AA zero-critical-violations gate

**Files:** `packages/testing/src/axe-setup.ts`, `apps/web-ecommerce/src/__tests__/axe.test.tsx`

**Behavior:**
- `runAxeTest(container: HTMLElement)` runs `axe.run(container)` and returns structured results
- Throws (fails test) if `violations.some(v => v.impact === 'critical')`
- Returns violations array for optional additional assertions
- Agency test files render minimal HTML fixtures (not full Next.js page — no network, no Payload) that replicate the semantic structure of P0 pages: `<main>`, `<nav>`, `<header>`, `<footer>`, skip link, `<h1>`, interactive buttons with labels

**Cases:**
- Fixture with proper semantic HTML → 0 critical violations → test passes
- Fixture missing `alt` on an `<img>` → axe flags `image-alt` as critical → test fails
- Fixture with no skip link → test assertion on skip link fails separately
- Fixture with button lacking aria-label → axe flags as critical → test fails

### Task 1: Create runAxeTest helper and update packages/testing

**File:** `packages/testing/src/axe-setup.ts`

```typescript
import axe from 'axe-core'

export interface AxeTestResult {
  violations: axe.Result[]
  passes: axe.Result[]
  incomplete: axe.Result[]
}

/**
 * Runs axe-core against a rendered HTML container and asserts zero critical violations.
 * Throws a Vitest-friendly error listing all critical violations if any are found.
 *
 * REQ-096: WCAG 2.2 AA CI gate — zero critical violations = ship condition.
 *
 * @param container - The HTMLElement to scan (typically from @testing-library/react `render`)
 * @returns Full axe results for additional assertions
 */
export async function runAxeTest(container: HTMLElement): Promise<AxeTestResult> {
  const results = await axe.run(container, {
    runOnly: {
      type: 'tag',
      values: ['wcag2a', 'wcag2aa', 'wcag21aa', 'wcag22aa', 'best-practice'],
    },
  })

  const critical = results.violations.filter(v => v.impact === 'critical')

  if (critical.length > 0) {
    const details = critical
      .map(v => `  [${v.id}] ${v.description}\n    Nodes: ${v.nodes.map(n => n.html).join(', ')}`)
      .join('\n')
    throw new Error(
      `axe-core found ${critical.length} critical WCAG violation(s):\n${details}`
    )
  }

  return {
    violations: results.violations,
    passes: results.passes,
    incomplete: results.incomplete,
  }
}
```

### Task 2: Create ecommerce axe test suite and vitest config

**File:** `apps/web-ecommerce/src/__tests__/axe.test.tsx`

This test renders self-contained HTML fixtures that replicate the semantic structure of P0 pages (home, about, services, blog, contact). It does NOT import actual page components (those need Payload CMS running). Instead it tests the HTML output structure that the pages produce, catching structural WCAG violations before full integration tests.

```typescript

/**
 * WCAG 2.2 AA — axe-core tests for web-ecommerce P0 page structures.
 * REQ-096: Zero critical violations = CI gate pass.
 *
 * These tests use HTML fixtures matching the semantic structure of Phase 8 P0 pages.

function createPageFixture(bodyContent: string): HTMLElement {

        <a href="/blog">Blog</a>

      <p>© 2026 MJ Ecommerce Agency. All rights reserved.</p>
    </footer>
  `
  return div
}
```
