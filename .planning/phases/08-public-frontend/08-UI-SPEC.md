## Design System

| Property | Value | Source |
|----------|-------|--------|
| Tool | none — custom `var(--mj-*)` token system | STATE.md: "45 React block components use `var(--mj-*)` tokens only — zero hex literals" |
| Preset | not applicable | No `components.json` — not a shadcn project |
| Component library | Custom 45-block library in `packages/ui/src/blocks/` | Phase 5 deliverable |
| Icon library | Lucide React (used across existing block components) | Codebase scan |
| Token import | `@import "@mjagency/ui/styles/theme.css"` in every agency `globals.css` | `apps/web-main/src/app/globals.css` |
| Theme resolution | 6-layer CSS cascade via `[data-agency="..."]` + `[data-theme="..."]` attributes | `packages/ui/tokens/` |
| FOUC prevention | Inline `<script>` as first child of `<head>` — reads `localStorage('mj-theme')` before paint | `08-PATTERNS.md` Pattern 2 |

**Token rule (mandatory, non-negotiable):** Every React component in Phase 8 must use ONLY `var(--mj-*)` CSS custom properties. Zero hex literals. Zero hardcoded OKLCH values. Zero inline `color:`, `background-color:`, `font-size:`, `margin:`, or `padding:` with raw values. Violations are caught by the Phase 4 AJV validator CI gate.

## Spacing Scale

Pre-populated from `packages/ui/tokens/layer-4-layout.css`. All values are multiples of 4.

| Token | CSS Variable | Value | Usage in Phase 8 |
|-------|-------------|-------|-----------------|
| xs | `var(--mj-space-1)` | 4px | Icon-to-label gaps, inline badge padding |
| sm | `var(--mj-space-2)` | 8px | Button internal padding-y, tag spacing |
| md | `var(--mj-space-4)` | 16px | Button padding-x, form field padding-x, default gap |
| lg | `var(--mj-space-6)` | 24px | Card padding (`--mj-card-padding`), layout gutter |
| xl | `var(--mj-space-8)` | 32px | Between-block spacing within a page section |
| 2xl | `var(--mj-space-12)` | 48px | Section padding-top / padding-bottom |
| 3xl | `var(--mj-space-16)` | 64px | Page hero top padding, major section breaks |
| 4xl | `var(--mj-space-20)` | 80px | Full-width section vertical rhythm |
| 5xl | `var(--mj-space-24)` | 96px | Hero section on desktop |

**Exceptions:**
- Touch targets (mobile nav links, CTA buttons on mobile): minimum 44px height — use `min-height: 44px` with padding adjustment. This is a WCAG 2.2 SC 2.5.8 requirement (Target Size Minimum).
- BlurHash placeholder containers: `aspect-ratio` preserves dimensions; no explicit spacing override needed.
- AI disclosure banner: `padding: var(--mj-space-2) var(--mj-space-4)` (compact alert pattern).

## Color

All values expressed as CSS custom properties only. No hex literals. No raw OKLCH in component code.

Per-agency brand colors are set via `[data-agency="..."]` attribute blocks in `packages/ui/styles/agencies.generated.css` — components never hard-code them.

### Color Contract (60/30/10)

| Role | CSS Variable | Value (default / brand.com) | Usage |
|------|-------------|----------------------------|-------|
| Dominant (60%) — surface | `var(--mj-color-bg-primary)` | `oklch(1 0 0)` (white) | Page backgrounds, card backgrounds, main surface |
| Secondary (30%) — structure | `var(--mj-color-bg-secondary)` | `oklch(0.98 0 0)` (near-white) | Alternating section backgrounds, nav background, footer background |
| Accent (10%) — brand | `var(--mj-color-brand-500)` | Agency-specific OKLCH (see below) | Primary CTA buttons, active nav links, focus rings, icon highlights, inline text links |
| Destructive | `var(--mj-color-error)` | `oklch(0.63 0.21 25)` (red) | Form validation error states ONLY — no other use |

**Accent reserved-for list (exhaustive):**
1. Primary CTA button background (`--mj-btn-bg-primary`)
2. Active / current nav link underline or indicator
3. Focus ring on all interactive elements (`outline-color: var(--mj-color-border-focus)`)
4. Inline text links (`color: var(--mj-color-text-link)`)
5. Icon accent state (`--mj-icon-accent`)
6. Section separator bar / decorative rule on hero blocks
7. Progress/loading indicator

**Accent must NOT appear on:** body text, card borders, section backgrounds, decorative illustrations (use `--mj-ill-primary` for illustrations).

## Interaction States

All interactive elements must declare all four states using `var(--mj-*)` tokens:

| State | Implementation |
|-------|---------------|
| Default | Base token values |
| Hover | Opacity 0.85 on brand background; `transition: var(--mj-btn-transition)` |
| Focus | `outline: 2px solid var(--mj-color-border-focus)` + `outline-offset: 2px` — visible on ALL interactive elements (WCAG 2.4.7) |
| Active/Pressed | Scale `0.98` via `transform: scale(0.98)` + `var(--mj-duration-fast)` |
| Disabled | `opacity: 0.5` + `cursor: not-allowed` + `pointer-events: none` |

**Focus ring rule:** Every `<a>`, `<button>`, `<input>`, `<textarea>`, `<select>` must show a visible focus ring using `var(--mj-color-border-focus)`. No `:focus` suppression without `:focus-visible` replacement.

## Image Pipeline Contract

| Requirement | Implementation | Constraint |
|-------------|---------------|------------|
| Format delivery | AVIF first, WebP fallback via Cloudflare Images variants | `formats: ['image/avif', 'image/webp']` in `next.config.mjs` |
| BlurHash placeholder | `placeholder="blur"` + `blurDataURL` from `BlurHashResult` | Computed at upload time, stored in DAM |
| Art direction | `MjImage` component in `packages/media/src/picture.tsx` | Wraps `next/image`; `sizes` prop mandatory |
| SVG policy | DOMPurify + SVGO sanitization at upload; `dangerouslyAllowSVG` banned | CLAUDE.md Rule 7 — zero exceptions |
| Remote patterns | `*.cloudflare.com` and `imagedelivery.net` only | No arbitrary external image hosts |
| Alt text | Required on every `MjImage` — empty alt only for decorative images | WCAG 1.1.1 (Level A) |

## Accessibility Contract (WCAG 2.2 AA)

| Criterion | Requirement | Verification |
|-----------|------------|-------------|
| 1.1.1 Non-text Content | All `<img>` / `MjImage` have meaningful `alt` | axe-core CI |
| 1.3.1 Info and Relationships | Semantic HTML: `<main>`, `<nav>`, `<header>`, `<footer>`, `<article>`, `<section>` | axe-core CI |
| 1.4.3 Contrast (Minimum) | 4.5:1 for body text, 3:1 for large text | axe-core CI + OKLCH token values verified in Phase 4 |
| 2.1.1 Keyboard | All interactive elements reachable via Tab; no keyboard traps | axe-core CI + Playwright e2e |
| 2.4.1 Bypass Blocks | Skip-to-main-content link as first focusable element on every page | `<a href="#main-content" className="sr-only focus:not-sr-only">Skip to content</a>` |
| 2.4.7 Focus Visible | Focus ring on all interactive elements using `var(--mj-color-border-focus)` | axe-core CI |
| 2.5.8 Target Size (Min) | Touch targets >= 44x44px on mobile | Manual + Playwright |
| 4.1.2 Name, Role, Value | `aria-label` on icon-only buttons; `aria-expanded` on accordions; `aria-current="page"` on nav | axe-core CI |

**axe-core CI gate:** Zero critical violations = ship condition. Run via `vitest` with `environment: 'jsdom'` per `08-PATTERNS.md` test pattern. CI fails if any `impact === 'critical'` violation found.

## Security Constraints (Phase 8 Specific)

| Rule | Implementation |
|------|---------------|
| No `dangerouslyAllowSVG` | Verified by `next.config.mjs` pattern; CI grep gate from Phase 1 |
